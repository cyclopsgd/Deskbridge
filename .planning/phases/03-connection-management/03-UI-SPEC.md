# Phase 3 — UI Design Contract: Connection Management

> Visual and interaction contract for the connection management phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> This is a **WPF desktop application** using WPF-UI (Fluent). All values are in device-independent pixels. All colours use DynamicResource tokens.
> Phase 3 **continues** the visual system established in Phase 2 (02-UI-SPEC.md). All spacing, typography, colour, and styling rules carry forward unchanged.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | WPF-UI 4.2.0 (Fluent Design) |
| Preset | Dark theme (`ui:ThemesDictionary Theme="Dark"`) with custom accent |
| Component library | WPF-UI controls + auto-restyled standard WPF controls (TreeView, TabControl, ContextMenu, TextBox, ComboBox) |
| Icon library | WPF-UI SymbolIcon (Fluent System Icons, `{ui:SymbolIcon Name24}`) |
| Font | Segoe UI Variable (Windows 11 system font, inherited from WPF-UI theme) |
| Layout engine | WPF Grid / DockPanel / StackPanel |
| MVVM | CommunityToolkit.Mvvm 8.4.2 (`[ObservableProperty]`, `[RelayCommand]`) |
| Base window | FluentWindow with Mica backdrop (established in Phase 2) |
| Dialog host | `ui:ContentDialogHost` (registered in MainWindow.xaml from Phase 2) |

## Interaction Contracts

### TreeView Selection (D-06)

```
Single-click:     Select item, deselect all others, update quick properties
Ctrl+click:       Toggle selection on clicked item, preserve other selections
Shift+click:      Select range from last-clicked to current item
Double-click:     On connection: open editor dialog. On group: toggle expand/collapse.
```

**Selection state is tracked by ConnectionTreeViewModel** (not the WPF TreeView.SelectedItem, since standard TreeView does not support multi-select). Use attached behavior or custom selection tracking.

### Context Menu (D-07)

```
Right-click on item:      Select item (if not already selected), show context menu
Right-click on selection: Show multi-item context menu (reduced items)
Right-click on empty:     Show: New Connection, New Group only
```

### Drag-Drop (D-05)

```
State machine:

[Idle] --mouse down on selected item--> [Pending] (record start position)
[Pending] --mouse move > 4px threshold--> [Dragging] (show drag ghost)
[Pending] --mouse up--> [Idle] (was a click, not a drag)
[Dragging] --hover over group--> [DragOverGroup] (highlight target group)
[Dragging] --hover between items--> [DragBetweenItems] (show insertion line)
[Dragging] --drop on group--> [Idle] (move items into group, persist)
[Dragging] --drop between items--> [Idle] (reorder items, persist)
[Dragging] --drop outside tree--> [Idle] (cancel drag)
[Dragging] --Escape key--> [Idle] (cancel drag)
```

**Drag threshold:** 4px from mouse-down position before entering drag mode (prevents accidental drags).
**Multi-item drag (D-06):** All selected items move together.

### F2 Rename

```
[Selected] --F2 key--> [Renaming] (TextBox replaces TextBlock)
[Renaming] --Enter--> [Selected] (commit rename, persist)
[Renaming] --Escape--> [Selected] (cancel rename, restore original name)
[Renaming] --focus lost--> [Selected] (commit rename, persist)
```

### Editor Dialog Lifecycle (D-10, D-12)

```
Open triggers:
  - Double-click connection in tree
  - "Edit" from context menu
  - "New Connection" from context menu or Ctrl+N
  - "New Group" from context menu

Save flow:
  1. Validate required fields (hostname for connection, name for group)
  2. If CredentialMode=Own and credential fields filled: store via ICredentialService
  3. Save model via IConnectionStore
  4. Close dialog, refresh tree
  5. Publish ConnectionCreated or ConnectionEdited event

Cancel flow:
  1. If unsaved changes: show "Discard changes?" dialog (only if user modified fields)
  2. Close dialog, no persistence
```

### Quick Properties Inline Edit (D-08)

```
Edit flow:
  1. User changes value in quick properties TextBox/ComboBox
  2. On focus-lost or Enter: validate and save immediately via IConnectionStore
  3. Tree item name updates if Name was changed
  4. No explicit save button -- changes are immediate
```
