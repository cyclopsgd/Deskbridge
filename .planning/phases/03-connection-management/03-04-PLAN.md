1. **Create TreeViewMultiSelectBehavior** at `src/Deskbridge/Behaviors/TreeViewMultiSelectBehavior.cs`:
   - Static class with attached DependencyProperty `EnableMultiSelect` (bool)
   - When enabled, attaches to TreeView's `PreviewMouseLeftButtonDown` event
   - Behavior logic (per D-06, UI-SPEC interaction contract):
     - **Plain click:** Deselect all items in ViewModel.SelectedItems, select clicked item only, set PrimarySelectedItem
     - **Ctrl+Click:** Toggle clicked item's IsSelected. If selected, add to SelectedItems. If deselected, remove from SelectedItems. Set PrimarySelectedItem to clicked item.
     - **Shift+Click:** Select range from _lastClickedItem to current item. Need a `GetFlatVisibleItems()` helper that walks the tree in display order (respecting expanded/collapsed groups). Select all items between the two indices. Set PrimarySelectedItem to clicked item.
   - Track `_lastClickedItem` for Shift+Click range calculation
   - Helper method `GetFlatVisibleItems(TreeView)`: Recursively enumerate all visible TreeItemViewModels by walking RootItems and expanded group Children. Returns `List<TreeItemViewModel>` in display order.
   - IMPORTANT: Use ViewModel's `IsSelected` property, NOT `TreeViewItem.IsSelected` (standard TreeView only allows single selection). The ItemContainerStyle already binds `TreeViewItem.IsSelected` to the ViewModel's `IsSelected` (set up in Plan 02).

2. **Create TreeViewDragDropBehavior** at `src/Deskbridge/Behaviors/TreeViewDragDropBehavior.cs`:
   - Static class with attached DependencyProperty `EnableDragDrop` (bool)
   - State machine (per D-05, UI-SPEC drag-drop contract):
     - **[Idle] -> mouse down on selected item -> [Pending]**: Record mouse start position
     - **[Pending] -> mouse move > 4px -> [Dragging]**: Start DoDragDrop. Use `DataObject.SetData(typeof(List<TreeItemViewModel>), selectedItems)` for in-process DnD (no BinaryFormatter per RESEARCH.md). Show drag ghost (Opacity=0.6 snapshot). For multi-item drag, show count badge.
     - **[Pending] -> mouse up -> [Idle]**: Was a click, not a drag
     - **[Dragging] -> drop on group -> [Idle]**: Move all dragged items into the target group. Call ViewModel's `MoveToGroup(targetGroupId)` which updates GroupId on connections, re-parents groups, and persists via IConnectionStore.
     - **[Dragging] -> Escape -> [Idle]**: Cancel drag
   - Visual indicators:
     - Drop ON group: Change target group TreeViewItem background to `SubtleFillColorSecondaryBrush` via Adorner or direct style
     - Drop BETWEEN items: 2px horizontal line in `SystemAccentColorPrimaryBrush` via AdornerLayer. Use a custom Adorner class `DropInsertionAdorner` that draws a 2px Line.
   - Drag threshold: 4px from mousedown position before entering drag mode
   - Cancel: Escape key during drag cancels the operation
