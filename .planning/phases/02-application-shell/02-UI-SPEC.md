# Phase 2 — UI Design Contract: Application Shell

> Visual and interaction contract for the Deskbridge application shell. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> This is a **WPF desktop application** using WPF-UI (Fluent). All values are in device-independent pixels. All colours use DynamicResource tokens.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | WPF-UI 4.2.0 (Fluent Design) |
| Preset | Dark theme (`ui:ThemesDictionary Theme="Dark"`) with custom accent |
| Component library | WPF-UI controls (`xmlns:ui="http://schemas.lepo.co/wpfui/2022/xaml"`) + auto-restyled standard WPF controls |
| Icon library | WPF-UI SymbolIcon (Fluent System Icons, `{ui:SymbolIcon Name24}`) |
| Font | Segoe UI Variable (Windows 11 system font, inherited from WPF-UI theme) |
| Layout engine | WPF Grid / DockPanel / StackPanel (NOT NavigationView -- decision D-01) |
| MVVM | CommunityToolkit.Mvvm 8.4.2 (`[ObservableProperty]`, `[RelayCommand]`) |
| Base window | FluentWindow with Mica backdrop, ExtendsContentIntoTitleBar="True" |

## Layout Dimensions

All values are fixed and derive from REFERENCE.md and CONTEXT.md locked decisions.

| Region | Height/Width | Notes |
|--------|-------------|-------|
| Title bar | 32px height | `ui:TitleBar Height="32"` (existing from Phase 1) |
| Icon rail | 36px wide | Left edge, full height between title bar and status bar |
| Slide-out panel | 240px wide | Fixed width (D-02), no GridSplitter, Visibility toggle (D-03) |
| Tab bar | 30px height | Always visible even with no tabs (D-07) |
| Status bar | 22px height | Bottom edge, full width |
| Viewport | Remaining space (`*`) | Must be airspace-safe -- no WPF elements may overlap |

### Grid Structure

The root Grid below the TitleBar row uses this structure:

```
Row 0: Auto      -- TitleBar (32px)
Row 1: *         -- Main content area
Row 2: 22        -- Status bar

Main content area (Row 1) column layout:
Column 0: 36     -- Icon rail
Column 1: Auto   -- Slide-out panel (240px or collapsed)
Column 2: *      -- Editor area

Editor area row layout:
Row 0: 30        -- Tab bar
Row 1: *         -- Viewport
```

## Spacing Scale

Declared values (multiples of 4, following WPF-UI Fluent Design spacing ramp):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gaps, tight inline padding, border thickness |
| sm | 8px | Between stacked controls, compact element margins |
| md | 12px | Card internal padding, panel content margins |
| lg | 16px | Section padding within panels, between control groups |
| xl | 24px | Major section breaks within panels |
| 2xl | 32px | Empty state content insets |
| 3xl | 48px | Not used in this phase |

**Exceptions:**
- Icon rail buttons: 36x36px touch targets (matches rail width)
- Tab close button: 16x16px hit area with 4px padding
- Status bar horizontal padding: 8px left and right

**WPF Margin/Padding convention:** `Margin="Left,Top,Right,Bottom"` or shorthand `Margin="Horizontal,Vertical"` or uniform `Margin="All"`.

## Typography

All sizes follow the WPF-UI / WinUI type ramp. FontSize is in device-independent pixels (WPF convention).

| Role | Size | Weight | Line Height | WPF-UI Token |
|------|------|--------|-------------|--------------|
| Caption | 12px | Regular (400) | 16px (1.33) | Used for status bar text, panel section headers |
| Body | 14px | Regular (400) | 20px (1.43) | Default for all UI text, panel content, tab labels |
| Body Strong | 16px | SemiBold (600) | 22px (1.375) | Panel titles, empty state heading |
| Subtitle | 20px | SemiBold (600) | 28px (1.4) | Empty viewport branding text only |

**Only 2 weights used throughout this phase:** Regular (400) and SemiBold (600).

**Font family:** Do not set explicitly. WPF-UI theme provides Segoe UI Variable via its resource dictionaries. All controls inherit it automatically.

## Color

### Theme Tokens (via DynamicResource)

All colours MUST use `{DynamicResource TokenName}` -- never hardcode hex values in XAML. The custom accent is applied via `ApplicationAccentColorManager.Apply()` in App.xaml.cs code-behind.

| Role | Token | Dark Theme Value | Usage |
|------|-------|-----------------|-------|
| **Dominant (60%)** | `ApplicationBackgroundColor` | #202020 | Window background, viewport background |
| **Secondary (30%)** | `ControlFillColorDefaultBrush` | Semi-transparent | Icon rail background, panel background, tab bar background |
| **Accent (10%)** | `SystemAccentColorPrimaryBrush` | #007ACC (custom) | See reserved list below |
| Text primary | `TextFillColorPrimaryBrush` | #FFFFFF | All primary text |
| Text secondary | `TextFillColorSecondaryBrush` | ~#999999 | Muted labels, shortcut hints, status bar secondary text |
| Text tertiary | `TextFillColorTertiaryBrush` | ~#666666 | Disabled/placeholder text |
| Border | `ControlStrokeColorDefaultBrush` | Semi-transparent | Panel right border, tab bar bottom border |
| Text on accent | `TextOnAccentFillColorPrimary` | #FFFFFF | Text on status bar, text on accent backgrounds |
| Subtle hover | `SubtleFillColorSecondaryBrush` | Semi-transparent | Icon rail hover state, tab hover state |

### Custom Accent Colours

Applied in App.xaml.cs via `ApplicationAccentColorManager.Apply()`:

| Name | Hex | Usage |
|------|-----|-------|
| Accent | #007ACC | Primary interactive accent |
| AccentHover | #1A8AD4 | Hover variant of accent |
| Error | #F44747 | Error states (defined as custom resource, not accent) |
| Success | #89D185 | Success states (defined as custom resource, not accent) |

These are declared as custom `SolidColorBrush` resources in App.xaml (after MergedDictionaries):

```xml
<SolidColorBrush x:Key="DeskbridgeErrorBrush" Color="#F44747" />
<SolidColorBrush x:Key="DeskbridgeSuccessBrush" Color="#89D185" />
```

### Accent Reserved For (explicit list)

The accent colour (#007ACC via `SystemAccentColorPrimaryBrush`) is used ONLY on these elements:

1. **Active tab indicator** -- 2px top border on the active tab item
2. **Active icon rail indicator** -- 2px left border on the active icon in the rail
3. **Status bar background** -- entire 22px status bar uses accent background
4. **Primary CTA buttons** -- `Appearance="Primary"` on `ui:Button` (not used in this phase)
5. **Focus rings** -- automatically applied by WPF-UI on keyboard focus

No other elements use accent colour. Icon rail icons, tab labels, and panel text are all neutral colours.

## Component Inventory

### 1. Icon Rail (Custom Control)

**Type:** Border containing DockPanel with StackPanels
**Width:** 36px
**Background:** `{DynamicResource ControlFillColorDefaultBrush}`
**Right border:** 1px `{DynamicResource ControlStrokeColorDefaultBrush}`

**Icons (3 total, decision D-05):**

| Position | Icon | SymbolIcon | Purpose |
|----------|------|------------|---------|
| Top, 1st | Connections | `{ui:SymbolIcon PlugConnected24}` | Toggle connections panel |
| Top, 2nd | Search | `{ui:SymbolIcon Search24}` | Toggle search panel |
| Bottom | Settings | `{ui:SymbolIcon Settings24}` | Toggle settings panel |

**Icon button style:**
- `ui:Button Appearance="Transparent"`
- Width="36", Height="36"
- Icon only, no text label
- Hover: `SubtleFillColorSecondaryBrush` background
- Active state: 2px left border in accent colour (`SystemAccentColorPrimaryBrush`)

**Interaction (decision D-04, VS Code toggle pattern):**
- Click icon: if panel closed, open panel showing that content
- Click same icon: close panel
- Click different icon: switch panel content (panel stays open)
- ViewModel tracks `ActivePanelMode` enum: `None`, `Connections`, `Search`, `Settings`

### 2. Slide-Out Panel (Custom Control)

**Type:** Border with DockPanel
**Width:** 240px fixed (decision D-02)
**Visibility:** Bound to `IsPanelVisible` (bool), instant snap (decision D-03)
**Background:** `{DynamicResource ControlFillColorDefaultBrush}`
**Right border:** 1px `{DynamicResource ControlStrokeColorDefaultBrush}`

**Panel header:**
- DockPanel.Dock="Top"
- TextBlock with panel title ("CONNECTIONS", "SEARCH", "SETTINGS")
- FontSize="11", FontWeight="Regular"
- Foreground: `{DynamicResource TextFillColorSecondaryBrush}`
- Margin="12,8"
- All caps (set in XAML via `Typography.Capitals="AllSmallCaps"` or hardcode uppercase string)

**Panel content (placeholder for Phase 2):**
- Connections: TextBlock "Connection tree will appear here" centered, muted
- Search: TextBlock "Search will appear here" centered, muted
- Settings: TextBlock "Settings will appear here" centered, muted

### 3. Tab Bar (Custom Control, decision D-06)

**Type:** Hand-built ItemsControl with custom ItemTemplate
**Height:** 30px, always visible (decision D-07)
**Background:** `{DynamicResource ControlFillColorDefaultBrush}`
**Bottom border:** 1px `{DynamicResource ControlStrokeColorDefaultBrush}`

**Tab item template:**
- Height: 30px
- Padding: 8px horizontal, 0px vertical
- Content: TextBlock (connection name, 14px, Regular) + Close button (16px X icon)
- Close button: `ui:SymbolIcon Dismiss16`, Appearance="Transparent", Visibility on hover
- Active tab: 2px top border in `{DynamicResource SystemAccentColorPrimaryBrush}`, `TextFillColorPrimaryBrush` text
- Inactive tab: no top border, `TextFillColorSecondaryBrush` text
- Hover (inactive): `SubtleFillColorSecondaryBrush` background
- Overflow: wrap ItemsControl in ScrollViewer with HorizontalScrollBarVisibility="Auto", VerticalScrollBarVisibility="Disabled"

**Interaction:**
- Left-click tab: switch active tab
- Middle-click tab: close tab (InputBinding on MouseBinding)
- Close button click: close tab
- Ctrl+Tab / Ctrl+Shift+Tab: cycle tabs (KeyBinding on MainWindow)
- Ctrl+W: close active tab (KeyBinding on MainWindow)

### 4. Status Bar

**Type:** Border with DockPanel
**Height:** 22px
**Background:** `{DynamicResource SystemAccentColorPrimaryBrush}` (accent-coloured, like VS Code)
**Text foreground:** `{DynamicResource TextOnAccentFillColorPrimary}`
**Font size:** 12px

**Layout:**
- Left side (DockPanel default): status text
- Right side (DockPanel.Dock="Right"): secondary info

**Placeholder content for Phase 2:**
- Left: "Ready"
- Right: "No active connection"

**Future content (phases 4+):**
- Left: hostname of active connection
- Right: resolution, latency, connection quality indicator

### 5. Viewport Area

**Type:** Border or Grid cell
**Size:** fills all remaining space (Grid column `*`, row `*`)
**Background:** `{DynamicResource ApplicationBackgroundColor}`
**Constraint:** No WPF elements may overlap this region (airspace rule)

**Empty state (decision D-08):**
- Centered vertically and horizontally
- Line 1: "Deskbridge" at 20px, SemiBold, `TextFillColorSecondaryBrush` (muted, not primary)
- Line 2: "Ctrl+N to create a connection" at 14px, Regular, `TextFillColorTertiaryBrush`
- Vertical gap between lines: 8px
- Disappears when first tab opens (bound to tab count)

## Interaction Contracts

### Icon Rail Toggle (D-04)

```
State machine: ActivePanelMode

[None] --click Connections--> [Connections] (panel opens)
[Connections] --click Connections--> [None] (panel closes)
[Connections] --click Search--> [Search] (panel switches)
[Search] --click Search--> [None] (panel closes)
[Search] --click Settings--> [Settings] (panel switches)
[Settings] --click Settings--> [None] (panel closes)
[Settings] --click Connections--> [Connections] (panel switches)
... (all permutations follow the same pattern)
```

**ViewModel properties:**
- `ActivePanelMode` (enum: None, Connections, Search, Settings)
- `IsPanelVisible` (computed: `ActivePanelMode != None`)
- `IsConnectionsActive`, `IsSearchActive`, `IsSettingsActive` (computed, drive icon rail active indicators)

**Commands:**
- `TogglePanelCommand(PanelMode mode)` -- single RelayCommand with parameter

### Panel Show/Hide (D-03)

- **Mechanism:** Grid column Width bound to `IsPanelVisible`
  - Visible: `Width="240"`
  - Hidden: `Width="0"` (or column uses `Auto` with panel `Visibility="Collapsed"`)
- **Animation:** None (instant snap, decision D-03)
- **Viewport behavior:** viewport area grows/shrinks to fill available space (column `*`)

### Keyboard Shortcuts (Phase 2 scope)

| Shortcut | Action | Scope |
|----------|--------|-------|
| Ctrl+Tab | Next tab | MainWindow KeyBinding |
| Ctrl+Shift+Tab | Previous tab | MainWindow KeyBinding |
| Ctrl+W | Close active tab | MainWindow KeyBinding |

**Note:** Ctrl+N (new connection), Ctrl+Shift+P (command palette), and other shortcuts are defined in later phases. Phase 2 wires the KeyBindings infrastructure only.

## ViewModel Contracts

### MainWindowViewModel Extensions

The existing `MainWindowViewModel` (from Phase 1) is extended with these properties:

```csharp
// Panel state
[ObservableProperty]
private partial PanelMode ActivePanelMode { get; set; } = PanelMode.None;

// Computed from ActivePanelMode
public bool IsPanelVisible => ActivePanelMode != PanelMode.None;
public bool IsConnectionsActive => ActivePanelMode == PanelMode.Connections;
public bool IsSearchActive => ActivePanelMode == PanelMode.Search;
public bool IsSettingsActive => ActivePanelMode == PanelMode.Settings;

// Tab state
[ObservableProperty]
private partial ObservableCollection<TabItemViewModel> Tabs { get; set; };

[ObservableProperty]
private partial TabItemViewModel? ActiveTab { get; set; };

// Status bar
[ObservableProperty]
private partial string StatusText { get; set; } = "Ready";

[ObservableProperty]
private partial string StatusSecondary { get; set; } = "No active connection";

// Commands
[RelayCommand]
private void TogglePanel(PanelMode mode) { ... }

[RelayCommand]
private void CloseTab(TabItemViewModel tab) { ... }

[RelayCommand]
private void SwitchTab(TabItemViewModel tab) { ... }
```

### PanelMode Enum

```csharp
public enum PanelMode { None, Connections, Search, Settings }
```

### TabItemViewModel

```csharp
public partial class TabItemViewModel : ObservableObject
{
    [ObservableProperty]
    private partial string Title { get; set; };

    [ObservableProperty]
    private partial bool IsActive { get; set; };

    public Guid ConnectionId { get; init; }
}
```
