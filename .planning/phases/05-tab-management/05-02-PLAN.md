<objective>
Wire the Plan 01 TabHostManager into the WPF shell: add the persistent `HostContainer` grid to MainWindow.xaml (D-04 — every WFH parented once, never removed until tab close), refactor MainWindow.xaml.cs's OnHostMounted/OnHostUnmounted/OnReconnectOverlayRequested to target HostContainer with per-tab WFH correlation (Tag = ConnectionId), add MainWindow.OnClosing → TabHostManager.CloseAllAsync sequential app-shutdown path (D-08), register ITabHostManager singleton in App.xaml.cs with eager startup resolution so event-bus subscriptions land before the first ConnectionRequestedEvent (Pitfall 7), update MainWindowViewModel's Tabs/ActiveTab/CloseTabCommand/SwitchTabCommand bodies to delegate to the service and add CloseOtherTabsCommand / CloseAllTabsCommand / ReopenLastClosedCommand (D-07 + D-16), and subscribe the ViewModel to TabSwitchedEvent so StatusText / StatusSecondary reflect the active tab (D-15 Phase 5 scope: hostname · state + resolution with em-dash placeholders for latency/quality).

Purpose: This is the hardest architectural change in Phase 5 — converting the viewport from single-slot to persistent multi-host. The `never re-parent` rule is load-bearing (WINFORMS-HOST-AIRSPACE §Option 3): re-parenting a WindowsFormsHost after AxHost init tears down HwndSource, breaks the RDP session, and can trigger UCEERR_RENDERTHREADFAILURE (dotnet/wpf #3719). The defensive `WR-02 loop` that currently removes all WFH children before mounting a new host (MainWindow.xaml.cs lines 93-99) MUST be deleted — it was valid for Phase 4's single-host model but directly contradicts D-04.

Output: Multi-host viewport with bullet-proof parenting invariants; ITabHostManager registered and eager-resolved; MainWindowViewModel commands delegate to the service; status bar binds to active tab; app-shutdown drains tabs sequentially; STA integration tests prove no re-parenting occurs under tab switch or drag-reorder; DI composition test passes (no circular dependency because Plan 01's publisher-side fix broke the cycle).
</objective>

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-03 | Denial of Service | OnClosing blocks on CloseAllAsync | mitigate | Wrapped in try/catch — if any per-host disconnect throws, the outer `.GetAwaiter().GetResult()` propagates; the top-level try/catch in MainWindow.OnClosing swallows (best-effort per existing Phase 4 pattern) so the window still closes.
