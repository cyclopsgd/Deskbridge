Purpose: Phase 5 is fundamentally a coordination refactor. Phase 4 already proved every dangerous COM primitive (siting, disposal, reconnect, airspace, GDI baseline). This plan converts the single-slot _active field into a dict-keyed service so subsequent plans (02, 03) can layer the persistent HostContainer + UI on top without touching Core lifecycle logic again. Every load-bearing rule from 05-RESEARCH.md (don't mutate BitmapPersistence at runtime, snapshot before iterating, never re-parent) is encoded into tests so regressions are caught immediately.

Output: TabHostManager + interface + Tab events shipped in Deskbridge.Core; ConnectionCoordinator slimmed down (loses _active, the duplicate-click guard, and the replacement branch); AirspaceSwapper holds per-host pre-drag visibility; ConnectionTreeViewModel.Connect is the single switch-to-existing chokepoint; full unit coverage for TAB-01, TAB-04, TAB-05, D-02, D-07, D-09, D-16, plus a regression test for AirspaceSwapper's WM_EXITSIZEMOVE restore.
