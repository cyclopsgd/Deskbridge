**FILE 3: CardAndPanelStyles.xaml**

Create `src/Deskbridge/Resources/CardAndPanelStyles.xaml` as a ResourceDictionary containing:

**Semantic fill brushes (FOUND-03 layered depth):**

The key insight is that panels and cards currently BOTH use `ControlFillColorDefaultBrush`, producing no visual layering. By assigning different WPF-UI fill tokens to panels vs cards, we create a three-layer depth hierarchy:
- Layer 0: Mica backdrop (the window itself)
- Layer 1: Panel fill (icon rail, slide-out panel, tab bar) -- `ControlFillColorDefault` (~6% white)
- Layer 2: Card fill (settings cards, property sections, toast containers) -- `ControlFillColorSecondary` (~8% white)
- Layer 3: Input fill -- WPF-UI built-in TextBox/ComboBox fill (handled by WPF-UI control templates)

Create these semantic brush aliases:

```xml
<SolidColorBrush x:Key="DeskbridgePanelBackgroundBrush"
                 Color="{DynamicResource ControlFillColorDefault}" />
<SolidColorBrush x:Key="DeskbridgeCardBackgroundBrush"
                 Color="{DynamicResource ControlFillColorSecondary}" />
```

**CRITICAL: Use `Color="{DynamicResource ControlFillColorDefault}"` (the Color key, NOT the Brush key). The `Color` property of `SolidColorBrush` expects a `Color` value, not a `Brush`. Using the `*Brush` key here would throw a runtime error.**

**Container styles (named styles with x:Key on Border):**

1. `CardContainerStyle` (TargetType=Border):
   - Background = `{DynamicResource DeskbridgeCardBackgroundBrush}`
   - BorderBrush = `{DynamicResource ControlStrokeColorDefaultBrush}`
   - BorderThickness = `1`
   - CornerRadius = `4`
   - Padding = `12`

2. `PanelBorderStyle` (TargetType=Border):
   - Background = `{DynamicResource DeskbridgePanelBackgroundBrush}`
   - BorderBrush = `{DynamicResource ControlStrokeColorDefaultBrush}`

3. `SeparatorStyle` (TargetType=Border):
   - Height = `1`
   - Background = `{DynamicResource ControlStrokeColorDefaultBrush}`
   - Margin = `{StaticResource FormFieldSpacing}` (references SpacingResources.xaml)

4. `ToastContainerStyle` (TargetType=Border):
   - Width = `360`
   - Margin = `0,0,0,8`
   - CornerRadius = `8`
   - Padding = `16`
   - Background = `{DynamicResource DeskbridgeCardBackgroundBrush}`
   - BorderBrush = `{DynamicResource ControlStrokeColorDefaultBrush}`
   - BorderThickness = `1`

5. `PanelHeaderStyle` (TargetType=Border):
   - Height = `28`
   - Background = `{DynamicResource SubtleFillColorSecondaryBrush}`

**APP.XAML UPDATE**

Update App.xaml to merge the three new ResourceDictionaries AFTER the WPF-UI dictionaries but BEFORE the inline resources:

```xml
<ResourceDictionary.MergedDictionaries>
    <!-- 1. WPF-UI theme dictionaries (MUST be first) -->
    <ui:ThemesDictionary Theme="Dark" />
    <ui:ControlsDictionary />
    <!-- 2. Deskbridge resource dictionaries (after WPF-UI, can override) -->
    <ResourceDictionary Source="Resources/SpacingResources.xaml" />
    <ResourceDictionary Source="Resources/TypographyStyles.xaml" />
    <ResourceDictionary Source="Resources/CardAndPanelStyles.xaml" />
</ResourceDictionary.MergedDictionaries>
```

**CRITICAL merge order:** SpacingResources first (TypographyStyles references its keys via StaticResource), TypographyStyles second, CardAndPanelStyles third (references SpacingResources keys via StaticResource). ThemesDictionary and ControlsDictionary MUST remain first.

Keep ALL existing inline resources (the 3 custom brushes, the converters) BELOW the MergedDictionaries as they are today. Do not move them.
