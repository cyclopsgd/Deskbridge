    **STAB-02 (per research Mitigation 1 -- bitmap snapshot during switch):**

    The root cause is that toggling WindowsFormsHost.Visibility from Collapsed to Visible triggers a WPF layout pass, but the Win32 child HWND repaints asynchronously. Between layout completion and the first WM_PAINT, the HWND shows black. The fix: capture the outgoing host's frame as a bitmap overlay, show it, then hide the overlay after the incoming host has painted.

    **Part A -- Add `SnapshotSingleHost` to AirspaceSwapper:**

    Add a public method that captures a single host's bitmap and shows its overlay Image, without affecting other hosts (unlike SnapshotAndHideAll which captures ALL):

    ```csharp
    /// <summary>
    /// Tab-switch airspace bypass: captures a bitmap snapshot of a single visible WFH
    /// and shows its overlay Image. Does NOT hide the WFH itself — the caller controls
    /// visibility. Returns true if the snapshot was taken successfully, false if the host
    /// was not registered, already collapsed, or capture failed.
    /// Call <see cref="ClearSingleHostSnapshot"/> after the incoming tab has painted.
    /// </summary>
    public bool SnapshotSingleHost(WindowsFormsHost host)
    {
        ObjectDisposedException.ThrowIf(_disposed, this);
        AssertDispatcher();

        if (!_hosts.TryGetValue(host, out var overlay))
            return false;

        if (host.Visibility == Visibility.Collapsed)
            return false;

        var snapshot = CaptureHwnd(host);
        if (snapshot is null)
            return false;

        overlay.Source = snapshot;
        overlay.Visibility = Visibility.Visible;
        return true;
    }

    /// <summary>
    /// Clears the snapshot overlay for a single host after the incoming tab has painted.
    /// </summary>
    public void ClearSingleHostSnapshot(WindowsFormsHost host)
    {
        ObjectDisposedException.ThrowIf(_disposed, this);
        AssertDispatcher();

        if (_hosts.TryGetValue(host, out var overlay))
        {
            overlay.Visibility = Visibility.Collapsed;
            overlay.Source = null;
        }
    }
    ```

    **Part B -- Refactor `SetActiveHostVisibility` in MainWindow.xaml.cs:**

    Replace the current implementation with a snapshot-aware version. The key sequence:
    1. Find the outgoing WFH (currently Visible) and the incoming WFH (target)
    2. If outgoing exists and differs from incoming, call `_airspace.SnapshotSingleHost(outgoing)` to freeze the outgoing frame as a bitmap overlay
    3. Set all WFH visibility as before (incoming = Visible, others = Collapsed)
    4. Schedule a deferred cleanup at `DispatcherPriority.Loaded` to clear the snapshot after the incoming WFH has painted

    ```csharp
    private void SetActiveHostVisibility(Guid activeId)
    {
        WindowsFormsHost? outgoing = null;
        WindowsFormsHost? incoming = null;

        // Identify outgoing (currently visible) and incoming (target) hosts
        foreach (var child in HostContainer.Children)
        {
            if (child is WindowsFormsHost wfh && wfh.Tag is Guid id)
            {
                if (id == activeId)
                    incoming = wfh;
                else if (wfh.Visibility == Visibility.Visible)
                    outgoing = wfh;
            }
        }

        if (incoming is null) return;
        if (outgoing == incoming) return;

        // Capture outgoing frame as bitmap before hiding (covers the black gap)
        bool snapshotTaken = false;
        if (outgoing is not null)
        {
            snapshotTaken = _airspace.SnapshotSingleHost(outgoing);
        }

        // Apply visibility: exactly one WFH Visible + IsEnabled, all others Collapsed
        foreach (var child in HostContainer.Children)
        {
            if (child is WindowsFormsHost wfh)
            {
                var isActive = wfh.Tag is Guid id && id == activeId;
                wfh.Visibility = isActive ? Visibility.Visible : Visibility.Collapsed;
                wfh.IsEnabled = isActive;
            }
            else if (child is ReconnectOverlay ov)
            {
                var isActive = ov.Tag is Guid id && id == activeId;
                ov.Visibility = isActive ? Visibility.Visible : Visibility.Collapsed;
            }
        }

        // Clear snapshot overlay after the incoming WFH has had a chance to paint
        if (snapshotTaken && outgoing is not null)
        {
            var hostToClean = outgoing;
            Dispatcher.InvokeAsync(() =>
            {
                _airspace.ClearSingleHostSnapshot(hostToClean);
            }, System.Windows.Threading.DispatcherPriority.Loaded);
        }
    }
    ```

    **Important constraints:**
    - Do NOT use `Visibility.Hidden` anywhere -- always use `Collapsed`. Hidden has been observed to tear down AxHost child HWNDs on some servers (e.g. xrdp). This is documented in the existing code comments.
    - The snapshot overlay Image is already registered per-host via `RegisterHost(wfh, overlayImage)`. The overlay is a sibling in the visual tree. The snapshot approach reuses this existing infrastructure.
    - Keep the existing `IsEnabled = isActive` pattern -- a hidden WFH that stays IsEnabled=true can capture keyboard input (documented in WINFORMS-HOST-AIRSPACE.md).
    - The `DispatcherPriority.Loaded` callback fires after layout and render, which gives the incoming WFH time to process its first WM_PAINT.
