**DepthToGuideLinesConverter.cs** -- Create a new converter in `src/Deskbridge/Converters/` that returns a collection of `double` values representing horizontal positions for vertical guide lines. The converter reuses the same visual-tree-walk depth calculation as `TreeViewItemIndentConverter`.

```csharp
using System.Globalization;
using System.Windows.Data;
using System.Windows.Media;

namespace Deskbridge.Converters;

/// <summary>
/// Returns a collection of horizontal positions (doubles) for vertical indent guide lines.
/// Each position is centered within its indent column: (i * IndentSize) + (IndentSize / 2).
/// A TreeViewItem at depth N produces N guide positions (for depths 0 through N-1).
/// Depth-0 items (root level) produce an empty collection (no guides needed).
/// </summary>
public sealed class DepthToGuideLinesConverter : IValueConverter
{
    public double IndentSize { get; set; } = 19.0;

    public object Convert(object value, Type targetType, object parameter, CultureInfo culture)
    {
        if (value is not TreeViewItem tvi)
            return Array.Empty<double>();

        int depth = 0;
        DependencyObject parent = VisualTreeHelper.GetParent(tvi);
        while (parent is not null)
        {
            if (parent is TreeViewItem) depth++;
            if (parent is TreeView) break;
            parent = VisualTreeHelper.GetParent(parent);
        }

        if (depth == 0)
            return Array.Empty<double>();

        var positions = new double[depth];
        for (int i = 0; i < depth; i++)
            positions[i] = (i * IndentSize) + (IndentSize / 2.0);

        return positions;
    }

    public object ConvertBack(object value, Type targetType, object parameter, CultureInfo culture)
        => throw new NotSupportedException();
}
```

Key design decisions:
- Returns `double[]` (not `Thickness[]`) -- each value is a Canvas.Left position for a 1px vertical line
- Uses same IndentSize=19 default as TreeViewItemIndentConverter for alignment
- Depth 0 (root items) returns empty array -- no guides at root level since there are no parents to indicate
- Centered within indent column: position = (i * 19) + 9.5, so lines align with the midpoint of each indent level's expander area

Add an `ItemsControl` inside the RowBorder's Grid to render vertical guide lines. The ItemsControl binds to the `DepthToGuideLinesConverter` which returns an array of horizontal positions. Each position renders a 1px-wide Rectangle stretching the full row height.

In the FullRowTreeViewItemStyle ControlTemplate, inside the `<Grid>` that is the child of `RowBorder`, add the indent guides ItemsControl BEFORE the AccentStripe Rectangle (so guides render behind content):

```xml
<Border x:Name="RowBorder" ...>
    <Border.Background>
        <SolidColorBrush x:Name="RowBrush" Color="Transparent" />
    </Border.Background>
    <Grid>
        <!-- TREE-03: Vertical indent guide lines -->
        <ItemsControl x:Name="IndentGuides"
                      ItemsSource="{Binding
                          Converter={StaticResource DepthToGuideLines},
                          RelativeSource={RelativeSource TemplatedParent}}"
                      IsHitTestVisible="False">
            <ItemsControl.ItemsPanel>
                <ItemsPanelTemplate>
                    <Canvas />
                </ItemsPanelTemplate>
            </ItemsControl.ItemsPanel>
            <ItemsControl.ItemContainerStyle>
                <Style TargetType="ContentPresenter">
                    <Setter Property="Canvas.Left" Value="{Binding}" />
                </Style>
            </ItemsControl.ItemContainerStyle>
            <ItemsControl.ItemTemplate>
                <DataTemplate>
                    <Rectangle Width="1"
                               Fill="{DynamicResource ControlStrokeColorDefaultBrush}"
                               Opacity="0.3"
                               Height="{Binding ActualHeight,
                                   RelativeSource={RelativeSource AncestorType=Border}}" />
                </DataTemplate>
            </ItemsControl.ItemTemplate>
        </ItemsControl>

        <!-- 2px left accent stripe (visible when selected) -->
        <Rectangle x:Name="AccentStripe" ... />
        ...
    </Grid>
</Border>
```

Key design decisions:
- `IsHitTestVisible="False"` -- guide lines do not interfere with mouse events (click, hover, drag)
- `Opacity="0.3"` -- subtle guides that do not compete with content (Research open question 1 recommendation: 30-40%)
- `Fill="{DynamicResource ControlStrokeColorDefaultBrush}"` -- uses the same brush as border hairlines between regions (design system: 1px hairlines use ControlStrokeColorDefaultBrush)
- `Canvas` panel with `Canvas.Left` binding positions each line at the exact horizontal offset returned by the converter
- `Height` binds to the RowBorder's `ActualHeight` so guides span the full row height
- `Width="1"` -- 1px hairline per design system specification for structural guides
- Guide lines appear behind AccentStripe and content (XAML ordering: first in Grid = lowest z-order)
- Root-level items show no guides (converter returns empty array for depth 0) -- correct behavior since root items have no parent hierarchy to visualize
