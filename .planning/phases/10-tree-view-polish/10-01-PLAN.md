**Part B: TREE-02 -- Animated hover transitions in FullRowTreeViewItemStyle ControlTemplate**

Replace the RowBorder's `Background="Transparent"` with a named `SolidColorBrush` so we can animate its Color property:

Change:
```xml
<Border x:Name="RowBorder"
        Background="Transparent"
```

To:
```xml
<Border x:Name="RowBorder"
```
And add the Background as a child element:
```xml
<Border x:Name="RowBorder"
        BorderThickness="0"
        MinHeight="28"
        CornerRadius="{DynamicResource ControlCornerRadius}"
        Margin="0,0,0,1"
        SnapsToDevicePixels="True">
    <Border.Background>
        <SolidColorBrush x:Name="RowBrush" Color="Transparent" />
    </Border.Background>
```

Then replace the ControlTemplate.Triggers section. Remove the old instant `IsMouseOver` Trigger and instant `DataTrigger` for selection. Replace with Storyboard-based animations:

```xml
<ControlTemplate.Triggers>
    <Trigger Property="IsExpanded" Value="True">
        <Setter TargetName="ItemsHost" Property="Visibility" Value="Visible" />
    </Trigger>
    <Trigger Property="HasItems" Value="False">
        <Setter TargetName="Expander" Property="Visibility" Value="Hidden" />
    </Trigger>

    <!-- TREE-02: Animated hover (150ms fade-in/out) -->
    <Trigger Property="IsMouseOver" Value="True">
        <Trigger.EnterActions>
            <BeginStoryboard x:Name="HoverEnter">
                <Storyboard>
                    <ColorAnimation
                        Storyboard.TargetName="RowBrush"
                        Storyboard.TargetProperty="Color"
                        To="{StaticResource SubtleFillColorSecondary}"
                        Duration="0:0:0.15" />
                </Storyboard>
            </BeginStoryboard>
        </Trigger.EnterActions>
        <Trigger.ExitActions>
            <BeginStoryboard x:Name="HoverExit">
                <Storyboard>
                    <ColorAnimation
                        Storyboard.TargetName="RowBrush"
                        Storyboard.TargetProperty="Color"
                        To="Transparent"
                        Duration="0:0:0.15" />
                </Storyboard>
            </BeginStoryboard>
        </Trigger.ExitActions>
    </Trigger>

    <!--
      Selection visual bound to VM IsSelected (multi-select support).
      Uses EnterActions/ExitActions so the animation overrides any
      running hover animation (WPF: later Storyboard wins).
      AccentStripe visibility is still via Setter (instant, correct).
    -->
    <DataTrigger Binding="{Binding IsSelected}" Value="True">
        <DataTrigger.EnterActions>
            <BeginStoryboard x:Name="SelectEnter">
                <Storyboard>
                    <ColorAnimation
                        Storyboard.TargetName="RowBrush"
                        Storyboard.TargetProperty="Color"
                        To="{StaticResource SystemAccentColorSecondary}"
                        Duration="0:0:0.08" />
                </Storyboard>
            </BeginStoryboard>
        </DataTrigger.EnterActions>
        <DataTrigger.ExitActions>
            <BeginStoryboard x:Name="SelectExit">
                <Storyboard>
                    <ColorAnimation
                        Storyboard.TargetName="RowBrush"
                        Storyboard.TargetProperty="Color"
                        To="Transparent"
                        Duration="0:0:0.15" />
                </Storyboard>
            </BeginStoryboard>
        </DataTrigger.ExitActions>
        <Setter TargetName="AccentStripe" Property="Visibility" Value="Visible" />
    </DataTrigger>
</ControlTemplate.Triggers>
```

Key design decisions:
- Selection animation is 80ms (faster than hover) for snappy feel
- Selection deselect is 150ms for smooth fade-out
- Selection color uses `StaticResource SystemAccentColorSecondary` (Color key, not Brush key -- per Phase 8 decision and Pitfall 1 from research)
- Hover uses `StaticResource SubtleFillColorSecondary` (Color key)
- `StaticResource` is acceptable in Storyboards since Deskbridge is dark-theme-only (Research assumption A2)
- AccentStripe visibility remains as instant Setter (animation not needed for a thin stripe toggle)
- DataTrigger for selection comes AFTER hover Trigger in XAML, and uses its own Storyboard which overrides the hover animation when both are active (Research Pitfall 2 mitigation)
