<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user action (context menu) → RDP session creation | Connect All can open many RDP sessions, consuming GDI handles. |
| BulkEditDialog input → persisted JSON store | Apply writes edited fields to disk via SaveBatch. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-23-06 | Denial of Service (self / GDI exhaustion) | ConnectAllAsync | mitigate | BULK-01 threshold gate: BulkConnectConfirmDialog shown when `projected > GdiWarningThreshold && ConfirmBeforeBulkOperations`; threshold read from user settings (BulkOperationsRecord), never hardcoded. Already-open tabs are SwitchTo (not re-opened), bounding handle growth. |
| T-23-07 | Information Disclosure | EditSelectedAsync logging + SaveBatch | mitigate | Log only ConnectionId/count via Serilog (`{Count}` / `{ConnectionId}`), never Hostname/Username/Domain/credential values. ApplyToModels (23-02) writes only non-secret fields; passwords never touched. |
| T-23-08 | Tampering / data loss | EditSelectedAsync persistence | mitigate | All-or-nothing: single atomic SaveBatch (.tmp rename, Phase 19); wrapped in try/catch — on failure persist nothing and show the error InfoBar. No per-item Save loop (IMP-04). |
| T-23-09 | Invalid Input | BulkEditDialog → model | mitigate | OnButtonClick gate (23-02) blocks Apply until Validate() passes (Port 1..65535, non-empty Hostname for enabled fields); divergent unchecked fields are never written. |
| T-23-10 | DoS (UI-thread crash) | credential/store path | accept | SaveBatch path is data-only (no credential API call — no auto-sync of Username/Domain to CredentialManager, documented decision A/Q#5). try/catch + log+swallow prevents UI-thread crash. |
</threat_model>
