# Phase 23 — UI Design Contract

> Visual and interaction contract for Bulk Operations UX (BULK-01, BULK-02, BULK-03).
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> Stack is WPF + WPF-UI 4.2.0 (Fluent dark). shadcn is not applicable — this is a desktop WPF app.
> Visual authority: `.claude/skills/deskbridge-design/` (README.md + colors_and_type.css). Every token below is pulled from that skill, `docs/DESIGN.md`, and the existing codebase. Do NOT invent a new design language.

---

## Scope — what this phase adds to the existing UI

This phase adds three interactions onto surfaces that already exist. It reuses, never replaces, established patterns.

| Feature | Surface (existing) | New element |
|---------|-------------------|-------------|
| BULK-01 Connect All | `ConnectionTreeControl.xaml` `GroupContextMenu` | New `Connect All` menu item + GDI confirmation dialog (conditional) |
| BULK-02 Disconnect All (group) | `ConnectionTreeControl.xaml` `GroupContextMenu` | New `Disconnect All` menu item (no confirmation — non-destructive) |
| BULK-03 Bulk Edit | `ConnectionTreeControl.xaml` `MultiSelectContextMenu` + new `BulkEditDialog` | New `Edit…` menu item + new `ui:ContentDialog` modal |

The multi-select tree behavior (`TreeViewMultiSelectBehavior`, `SelectedNodes`) already exists from Phase 3/15 and is the source of selection for BULK-03. No new selection mechanics.

## Spacing Scale

Deskbridge uses the **4px Fluent grid** (a superset of the 8-point scale — 4, 8, 12, 16, 24, 32 are all in active use). Reuse existing shared resources: `DialogContentMargin`, `FormFieldSpacing`.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-text gap in menu items and rows, accent-stripe inset |
| sm | 8px | Compact stacking between bulk-edit field rows, card-to-card gap |
| (grid) | 12px | Card internal padding, button padding (`12,6`), dialog content margin |
| md | 16px | Default dialog body padding, card internal padding |
| lg | 24px | Section spacing inside the dialog |
| xl | 32px | Reserved — not expected in this phase |

Exceptions: Fixed chrome sizes are mandatory and override the grid where listed — context-menu/row `MinHeight` **28px**, status dot **8×8**, menu/button icons **24px** (inline 16px), per-field checkbox column **Auto** (~20px target). Dialog `DialogMaxWidth` **520px** for `BulkEditDialog` (between GroupEditor 480 and ConnectionEditor 560 — bulk edit has the per-field checkbox column so it needs slightly more than the group editor).

---

## Typography

Segoe UI Variable, WinUI ramp, weights **400 and 600 only**. No 700, no italics. Reuse existing styles: `SectionLabelStyle` (11px uppercase), `FieldLabelStyle` / `FieldLabelCompactStyle` (12px), `BodyStyle` (14px), `ErrorTextStyle`.

| Role | Size | Weight | Line Height | Used for |
|------|------|--------|-------------|----------|
| Section label | 11px | 600 SemiBold | default | `FIELDS TO EDIT` header in bulk-edit dialog; multi-select count header (uppercase) |
| Caption / field label | 12px | 400 | default | Field labels in bulk-edit grid, divergent-value hint text, menu shortcut hints |
| Body | 14px | 400 | default | Menu item labels, field input text, dialog body sentence, button labels |
| Dialog body / card title | 16px | 400 (600 for title) | ~1.4 | Dialog confirmation sentence (GDI warning), card titles |
| Dialog title | 20px | 600 SemiBold | ~1.2 | `BulkEditDialog` title bar (WPF-UI ContentDialog `Title`); GDI confirm dialog title |

Heading line-height ~1.2, body ~1.4–1.5 — WPF-UI ContentDialog defaults; do not override.

## Color

Dark-first Windows 11 Fluent, all via `DynamicResource`. Greyscale on Mica; the only custom hexes are the three Deskbridge semantic brushes. Accent appears in **exactly three places app-wide** and this phase adds none.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `ApplicationBackgroundBrush` (`#202020` dark) / Mica | Dialog and menu surfaces, viewport |
| Secondary (30%) | `ControlFillColorDefaultBrush` | Dialog content cards, menu background, input fill |
| Accent (10%) | `SystemAccentColorPrimaryBrush` / `SystemAccentColorSecondaryBrush` | NOT introduced by this phase. Selected tree rows already use the 2px accent stripe + secondary fill; bulk edit inherits this from the existing multi-select highlight. Primary dialog buttons use WPF-UI's built-in Primary appearance (accent-tinted) — this is the established pattern from ConnectionEditorDialog. |
| Destructive | `DeskbridgeErrorBrush` (`#F44747`) | `Delete` menu items only (already present). Bulk operations in this phase are NOT destructive — no red. |
| Warning | `DeskbridgeWarningBrush` (`#FFCC02`) | GDI confirmation dialog: `ui:InfoBar` `Severity="Warning"` + `Warning24` icon. Matches the existing 15-session snackbar treatment. |

Accent reserved for: status-bar background, 2px left stripe on selected tree rows, 2px top border on active tab, and WPF-UI Primary-button tint. **This phase introduces no new accent usage.**

## Copywriting Contract

Voice: terse, technical, Title Case on actions, sentence case in dialog bodies, UPPERCASE on section labels, no emoji, `…` on any action opening a follow-up. Reuses the exact GDI-warning wording already shipped in `TabHostManager.FireGdiWarningIfCrossingThreshold` for consistency.

| Element | Copy |
|---------|------|
| Group menu — Connect All (BULK-01) | `Connect All` · icon `Play24` · placed above the New Connection / New Group block in `GroupContextMenu`, after a `Separator`. No ellipsis when count is within threshold (acts immediately); gains a confirmation only when over threshold. |
| Group menu — Disconnect All (BULK-02) | `Disconnect All` · icon `PlugDisconnected24` (matches existing palette command) · only enabled when the group has ≥1 active session. |
| Multi-select menu — Edit (BULK-03) | `Edit…` · icon `Edit24` · placed at top of `MultiSelectContextMenu`, above `Move to…`. Ellipsis required (opens dialog). |
| Multi-select count header | `{N} connections selected` (existing `MultiSelectHeader` MenuItem, disabled, SemiBold). Singular: `1 connection selected`. |
| GDI confirm dialog — title | `Connect all connections?` |
| GDI confirm dialog — body | `This will open {N} sessions. {threshold}+ active sessions may degrade performance. Continue?` (mirrors the shipped snackbar phrasing "performance may degrade") |
| GDI confirm dialog — buttons | Primary: `Connect All` · Close: `Cancel` (WPF-UI ContentDialog `PrimaryButtonText` / `CloseButtonText`) |
| Bulk-edit dialog — title | `Edit {N} connections` (e.g. `Edit 4 connections`) |
| Bulk-edit dialog — section label | `FIELDS TO EDIT` |
| Bulk-edit dialog — divergent value hint | placeholder text on a field whose selected connections differ: `Multiple values` (shown as `PlaceholderText`, tertiary text colour; the field is left blank until the user enables + edits it) |
| Bulk-edit dialog — per-field enable | unlabeled `CheckBox` in left column; tooltip `Apply this field to all selected connections` |
| Bulk-edit dialog — buttons | Primary: `Apply` (enabled only when ≥1 field checked) · Close: `Cancel` |
| Empty state | Not applicable — these actions are unavailable when no group/selection exists (menu items simply don't appear). No empty-state surface is added. |
| Error state | Bulk edit save failure: `ui:InfoBar` `Severity="Error"`, title `Couldn't apply changes`, message `{n} of {N} connections could not be updated. No changes were saved.` (batch is atomic via SaveBatch — all-or-nothing). |
| Disconnect All toast (success) | Reuse existing toast infra; no new copy required (closures fire existing per-session ConnectionClosedEvent toasts, or suppress — planner decides; default: silent, matching palette "Disconnect All"). |

Destructive actions in this phase: **none.** Connect All and Disconnect All are reversible/operational, not data-destructive — they get a warning (GDI) or no confirmation (disconnect), never a red destructive confirmation. The existing `Delete` items are unchanged.

## Component Inventory (executor reference)

| Component | Type | Pattern source |
|-----------|------|----------------|
| `Connect All` / `Disconnect All` menu items | native `MenuItem` with `ui:SymbolIcon` Icon, `Command` + `CommandParameter` via `PlacementTarget.DataContext` | existing `GroupContextMenu` in `ConnectionTreeControl.xaml` |
| `Edit…` multi-select menu item | native `MenuItem` | existing `MultiSelectContextMenu` |
| GDI confirmation dialog | `ui:ContentDialog`, `DialogMaxWidth="420"`, `PrimaryButtonAppearance="Primary"`, `DefaultButton="Primary"`, body = sentence + `ui:InfoBar Severity="Warning"` | `ConnectionEditorDialog.xaml` header pattern; InfoBar usage from same file |
| `BulkEditDialog` | `ui:ContentDialog`, `DialogMaxWidth="520"`, `Title`, `PrimaryButtonText="Apply"`, `CloseButtonText="Cancel"`, `IsPrimaryButtonEnabled="{Binding CanApply}"`, `BasedOn` ContentDialog style | `ConnectionEditorDialog.xaml` / `GroupEditorDialog.xaml` |
| Bulk-edit field rows | `Grid` with 3 columns: `Auto` (CheckBox) / `72` (label) / `*` (input) — mirrors the existing quick-properties grid column widths (`72`/`*`) plus a checkbox column | `ConnectionTreeControl.xaml` quick-properties grids |
| Field inputs | `ui:TextBox` (Name/Hostname/Port/Username/Domain), `ComboBox` (Credential mode, Group) with explicit `ItemTemplate` to avoid the "- - -" SelectionBoxItem glyph trap | `ConnectionEditorDialog.xaml` (see its inline comments on ItemTemplate) |
| Per-field enable | native `CheckBox`, `IsChecked` two-way to a per-field `IsEnabled` VM flag; input `IsEnabled` bound to the same flag | new |

Bulk-edit editable fields (BULK-03 "shared/divergent"): Name is **excluded** (each connection keeps its unique name — bulk-renaming is nonsensical). Editable set: **Hostname, Port, Credential mode, Username, Domain, Group**. A field whose values are identical across the selection pre-fills with that shared value; a divergent field shows `Multiple values` placeholder and stays blank until enabled.

## Interaction Contract

- **Connect All:** right-click group → `Connect All`. If `(active sessions + connections in group) ≤ GdiWarningThreshold` OR `ConfirmBeforeBulkOperations == false`, connect immediately. Otherwise show the GDI confirmation dialog first. Settings come from `BulkOperationsRecord` (`GdiWarningThreshold`, `ConfirmBeforeBulkOperations`) — already in `AppSettings`.
- **Disconnect All:** right-click group → `Disconnect All`. No confirmation (non-destructive, instantly reversible by reconnecting). Item disabled when the group has no active sessions.
- **Bulk Edit:** select ≥2 connections → right-click → `Edit…` → modal `BulkEditDialog`. Each field has a checkbox; only checked fields are written. `Apply` is disabled until ≥1 checkbox is checked. On `Apply`, write via the Phase 19 `SaveBatch` API (single atomic write); publish `ConnectionDataChangedEvent`. On failure, show the error InfoBar and persist nothing.
- **Modal hosting:** all dialogs are WPF-UI `ContentDialog`s. Per `docs/WPF-UI-PITFALLS.md` and the shipped airspace fix (quick task 260419-efs), ensure ContentDialog z-orders correctly over an active RDP viewport. Reuse the established `ContentPresenter`/`DialogHost` hosting used by `ConnectionEditorDialog`.
