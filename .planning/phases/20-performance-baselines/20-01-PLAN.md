    ```csharp
    public abstract record TreeNode(Guid Id, string Name, int SortOrder, int Depth);

    public sealed record GroupNode(
        Guid Id,
        string Name,
        int SortOrder,
        int Depth,
        Guid? ParentGroupId,
        IReadOnlyList<TreeNode> Children) : TreeNode(Id, Name, SortOrder, Depth);

    public sealed record ConnectionNode(
        Guid Id,
        string Name,
        int SortOrder,
        int Depth,
        Guid? GroupId,
        string Hostname) : TreeNode(Id, Name, SortOrder, Depth);
    ```

    NOTE: Records are immutable. The builder constructs them with final depth values after computing the full tree. Do NOT try to mutate Depth after construction -- build the tree structure first (without depth), then reconstruct with correct depth in a final pass.

    **Step 2: Create `src/Deskbridge.Core/Services/ConnectionTreeBuilder.cs`**

    Static class in namespace `Deskbridge.Core.Services`. Single public method:

    ```csharp
    public static IReadOnlyList<TreeNode> Build(
        IReadOnlyList<ConnectionModel> connections,
        IReadOnlyList<ConnectionGroup> groups)
    ```

    Algorithm (port from ConnectionTreeViewModel.BuildTree lines 425-556):

    1. Build group lookup: `Dictionary<Guid, (ConnectionGroup group, List<object> children)>` -- track children as mixed list initially
    2. Cycle detection: For each group with ParentGroupId != null, walk the parent chain using a `HashSet<Guid>` visited set. If a cycle is detected (visited.Add returns false), mark the group for root promotion. Log warning via `Serilog.Log.Warning("Cycle detected...")`.
    3. Nest groups: Non-cyclic groups with valid ParentGroupId get added to parent's children list. Others go to root.
    4. Place connections: Each connection goes into its GroupId's children list, or root if GroupId is null or points to a nonexistent group.
    5. Sort siblings: At each level, sort by SortOrder ascending, then Name case-insensitive (`StringComparer.OrdinalIgnoreCase`) as tiebreaker. Groups and connections sort together (not groups-first).
    6. Assign depths and construct records: Recursively traverse the sorted tree, constructing `GroupNode`/`ConnectionNode` records with the correct Depth value. Root items get Depth=0, children get parent's Depth+1.

    Return `IReadOnlyList<TreeNode>` of root-level items.

    ```csharp
    private void BuildTree()
    {
        var connections = _connectionStore.GetAll();
        var groups = _connectionStore.GetGroups();

        // Build a lookup so the mapper can populate Port/Username/Domain/CredentialMode
        // from the original ConnectionModel (ConnectionNode only carries Id/Name/Hostname/GroupId/SortOrder/Depth)
        var connectionLookup = connections.ToDictionary(c => c.Id);

        var tree = ConnectionTreeBuilder.Build(connections, groups);
        var rootItems = MapToViewModels(tree, connectionLookup);

        _fullTree = rootItems;
        RootItems = new ObservableCollection<TreeItemViewModel>(rootItems);

        // TREE-01: restore status dots after tree rebuild

        {
            var treeItem = FindItemById(RootItems, connId) as ConnectionTreeItemViewModel;
            if (treeItem is not null)
                treeItem.ConnectionState = state;
        }
    }
    ```

    **Step 2: Add private `MapToViewModels` method**

    Signature: `private ObservableCollection<TreeItemViewModel> MapToViewModels(IReadOnlyList<TreeNode> nodes, IReadOnlyDictionary<Guid, ConnectionModel> connectionLookup)`

    The `connectionLookup` parameter is required because `ConnectionNode` is a lightweight record carrying only fields needed for tree structure (Id, Name, Hostname, GroupId, SortOrder, Depth). The ViewModel needs additional fields (Port, Username, Domain, CredentialMode) that exist on the original `ConnectionModel`. Looking them up by Id avoids bloating the pure tree record with UI-specific fields.

    ```csharp
    private ObservableCollection<TreeItemViewModel> MapToViewModels(
        IReadOnlyList<TreeNode> nodes,
        IReadOnlyDictionary<Guid, ConnectionModel> connectionLookup)
    {
        var result = new ObservableCollection<TreeItemViewModel>();
        foreach (var node in nodes)
        {
            switch (node)
            {
                case GroupNode g:
                    var groupVm = new GroupTreeItemViewModel
                    {
                        Id = g.Id,
                        Name = g.Name,
                        ParentGroupId = g.ParentGroupId,
                        SortOrder = g.SortOrder,
                        IsExpanded = true,
                        HasCredentials = _credentialService.HasGroupCredentials(g.Id),
                        Depth = g.Depth
                    };
                    foreach (var child in MapToViewModels(g.Children, connectionLookup))
                        groupVm.Children.Add(child);
                    result.Add(groupVm);
                    break;
                case ConnectionNode c:
                    var connVm = new ConnectionTreeItemViewModel
                    {
                        Id = c.Id,
                        Name = c.Name,
                        Hostname = c.Hostname,
                        GroupId = c.GroupId,
                        SortOrder = c.SortOrder,
                        Depth = c.Depth
                    };
                    // Populate fields not carried by ConnectionNode from the original model
                    if (connectionLookup.TryGetValue(c.Id, out var model))
                    {
                        connVm.Port = model.Port;
                        connVm.Username = model.Username;
                        connVm.Domain = model.Domain;
                        connVm.CredentialMode = model.CredentialMode;
                    }
                    result.Add(connVm);
                    break;
            }
        }
        return result;
    }
    ```

    CRITICAL: The original BuildTree() (lines 443-454) populated Port, Username, Domain, and CredentialMode on every ConnectionTreeItemViewModel. Without the lookup, these fields default to 3389, null, null, Inherit -- which would silently break connection functionality (wrong ports, missing credentials). The `connectionLookup.TryGetValue` pattern is defensive but the lookup should always succeed since the same connections list was passed to Build().
