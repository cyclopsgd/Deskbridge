## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| App -> GitHub API | Update check fetches release info from GitHub via HTTPS |
| GitHub -> App | Downloaded update packages applied to local filesystem |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | Update packages | accept | Velopack handles package signing; GitHub HTTPS provides transport security. No custom signing configured (CONTEXT.md does not require it). |
| T-07-02 | Denial of Service | UpdateService.CheckForUpdatesAsync | mitigate | CancellationToken support + try/catch with Log.Warning — failed check does not block app startup or throw to user |
| T-07-03 | Information Disclosure | Update settings | mitigate | UseBetaChannel preference stored in settings.json (non-sensitive). No credentials stored in update config. |
| T-07-04 | Elevation of Privilege | ApplyUpdatesAndRestart | mitigate | Confirmation dialog requires explicit user action before restart. No auto-restart without consent. |
</threat_model>
