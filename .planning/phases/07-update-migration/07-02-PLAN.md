  <action>
    1. Read the existing `.github/workflows/build.yml` in full.

    2. Add `tags: ['v*.*.*']` to the `on.push` trigger alongside the existing `branches: [main]`:
       ```yaml
       on:
         push:
           branches: [main]
           tags: ['v*.*.*']
         pull_request:
           branches: [main]
         workflow_dispatch:
       ```

    3. Add a new `release` job after the existing `build` job. The release job:
       - Runs only on tag pushes: `if: startsWith(github.ref, 'refs/tags/v')`
       - Needs the build job to pass first: `needs: build`
       - Runs on `windows-latest`
       - Has `permissions: contents: write` (needed for creating GitHub Releases)

    4. Release job steps:

       a. `actions/checkout@v4`

       b. `actions/setup-dotnet@v4` with `dotnet-version: 10.0.x`

       c. **Extract version from tag:**
          ```yaml
          - name: Extract version from tag
            id: version
            shell: bash
            run: echo "version=${GITHUB_REF_NAME#v}" >> $GITHUB_OUTPUT
          ```

       d. **Determine channel** (stable vs beta based on SemVer2 prerelease suffix):
          ```yaml
          - name: Determine channel
            id: channel
            shell: bash
            run: |
              if [[ "${{ steps.version.outputs.version }}" == *"-"* ]]; then
                echo "channel=beta" >> $GITHUB_OUTPUT
                echo "prerelease=true" >> $GITHUB_OUTPUT
              else
                echo "channel=stable" >> $GITHUB_OUTPUT
                echo "prerelease=false" >> $GITHUB_OUTPUT
              fi
          ```

       e. **Publish** (self-contained win-x64, version from tag):
          ```yaml
          - name: Publish
            run: >
              dotnet publish src/Deskbridge/Deskbridge.csproj
              --configuration Release
              --runtime win-x64
              --self-contained true
              -p:Version=${{ steps.version.outputs.version }}
              --output publish
          ```
          NOTE: Do NOT use `-p:PublishSingleFile=true` for the release build — Velopack vpk packs the output directory. SingleFile creates a self-extracting exe that Velopack cannot delta-compress efficiently.

       f. **Install vpk CLI:**
          ```yaml
          - name: Install vpk
            run: dotnet tool install -g vpk
          ```

       g. **Download previous release** (for delta computation, continue-on-error for first release per Pitfall 3):
          ```yaml
          - name: Download previous release
            continue-on-error: true
            run: >
              vpk download github
              --repoUrl https://github.com/${{ github.repository }}
              --channel ${{ steps.channel.outputs.channel }}
              --token ${{ secrets.GITHUB_TOKEN }}
          ```

       h. **Pack with vpk:**
          ```yaml
          - name: Pack
            run: >
              vpk pack
              -u Deskbridge
              -v ${{ steps.version.outputs.version }}
              -p publish
              --packTitle "Deskbridge"
              --channel ${{ steps.channel.outputs.channel }}
          ```

       i. **Upload to GitHub Release:**
          ```yaml
          - name: Upload to GitHub Release
            run: >
              vpk upload github
              --repoUrl https://github.com/${{ github.repository }}
              --channel ${{ steps.channel.outputs.channel }}
              --token ${{ secrets.GITHUB_TOKEN }}
              --publish
              --releaseName "Deskbridge ${{ steps.version.outputs.version }}"
              --tag ${{ github.ref_name }}
              ${{ steps.channel.outputs.prerelease == 'true' && '--pre' || '' }}
          ```

    5. The top-level `permissions: contents: read` stays for the build job. The release job overrides with `permissions: contents: write` at the job level.

    6. Verify the YAML is valid by checking indentation carefully. The release job is a sibling of the build job under the `jobs` key.
  </action>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| GitHub Actions -> GitHub Releases | Workflow writes release assets using GITHUB_TOKEN |
| vpk download -> GitHub API | Downloads prior release assets for delta computation |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-05 | Elevation of Privilege | GITHUB_TOKEN permissions | mitigate | `contents: write` scoped to release job only; build job retains `contents: read`. Token is short-lived and workflow-scoped. |
| T-07-06 | Tampering | Release artifacts | accept | GitHub Releases assets served over HTTPS. Velopack package integrity verified by the client's UpdateManager. No custom signing configured (deferred to user decision). |
| T-07-07 | Information Disclosure | GITHUB_TOKEN in logs | mitigate | Token passed via `--token ${{ secrets.GITHUB_TOKEN }}` which GitHub Actions masks automatically in logs. |
</threat_model>
