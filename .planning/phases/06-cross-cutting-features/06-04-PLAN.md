<objective>
Final Phase 6 plan — deliver the app-security surface: PBKDF2 master password with auth.json (SEC-01), a full-window opaque lock overlay with Pitfall 5 airspace mitigation (SEC-02), a DispatcherTimer idle lock with Pitfall 6 AxHost-input filter (SEC-03), Ctrl+L manual lock AND SystemEvents.SessionSwitch auto-lock with Pattern 9 strong-ref + Dispatcher marshal (SEC-04), Window.StateChanged Minimized lock when configured (SEC-05), and the real CrashDialog UI backing the Plan 06-01 stub (LOG-04 completion). This plan is LAST (D-20 Wave 4) because its surface is heaviest and every other Phase 6 feature must be testable without lock-screen friction during development. Non-autonomous because a Wave 0 visual spike is required (Q4) and three manual UAT files cover behaviours that cannot be unit-tested (suspend/resume, airspace, real SessionSwitch).

Purpose: Security must not be the glue that hides bugs — the attack model (D-17, CONTEXT Attack-model note) is clear: master password gates the Deskbridge UI, not the Windows Credential Manager secrets. Every mitigation in this plan targets a specific named threat: T-06-03 brute-force (PBKDF2 600k + FixedTimeEquals), T-06-04 lock bypass via airspace leak (HostContainer Visibility capture/restore per Pitfall 5 Option A), T-06-05 password-in-memory (scoped tightly to LockOverlayViewModel; cleared on successful unlock per SecureString-avoided pattern). The Wave 0 visual spike (Q4) proves the ContentDialog chrome renders correctly without a close-X button before we invest in the full lock flow. The three UAT files catch the behaviours that pure unit tests can't: a real Windows suspend/resume cycle (Pitfall 4), a real Windows session lock event (A4), and real RDP-session rendering underneath the lock overlay (Pitfall 5 validation).

Output: MasterPasswordService in Core; LockOverlayDialog + LockOverlayViewModel + 4 new Services (IdleLockService, SessionLockService, AppLockController, CrashDialog UI wired into CrashHandler); IAppLockState.Lock/Unlock populated from AppLockController; KeyboardShortcutRouter Ctrl+L added; MainWindow wiring for Window.StateChanged + app-startup lock; settings UI already shipped in Plan 06-02 (SecuritySettingsRecord) gets BINDING wired to MainWindowViewModel; three UAT checklists in tests/uat/; full unit coverage for PBKDF2, FixedTimeEquals, idle filter, SessionSwitch marshal/unsub, AppLockController state transitions, Ctrl+L routing.

   {
       private readonly DispatcherTimer _timer;
       private readonly IEventBus _bus;
       private readonly PreProcessInputEventHandler _handler;   // strong ref (Pitfall 1 principle applied here too)
       private bool _disposed;

       public IdleLockService(IEventBus bus, SecuritySettingsRecord security)
       {
           _bus = bus;
           _timer = new DispatcherTimer(DispatcherPriority.Normal,
               System.Windows.Application.Current?.Dispatcher ?? Dispatcher.CurrentDispatcher)
           {
               Interval = TimeSpan.FromMinutes(security.AutoLockTimeoutMinutes),
           };
           _timer.Tick += (_, _) => { _timer.Stop(); _bus.Publish(new AppLockedEvent(LockReason.Timeout)); };
           _handler = (_, e) => HandleInput(e);
           InputManager.Current.PreProcessInput += _handler;
           _timer.Start();
       }

       // Test seam + Pitfall 6 filter.
       internal void HandleInput(PreProcessInputEventArgs e)
       {
           if (_disposed) return;
           var src = e.StagingItem?.Input?.Source as DependencyObject;
           if (src is not null && FindAncestor<WindowsFormsHost>(src) is not null) return;
           _timer.Stop();
           _timer.Start();
       }

       private static T? FindAncestor<T>(DependencyObject d) where T : DependencyObject
       {
           while (d is not null)
           {
               if (d is T t) return t;
               d = VisualTreeHelper.GetParent(d) ?? LogicalTreeHelper.GetParent(d);
           }
           return null;
       }

       public void Dispose()
       {
           _disposed = true;
           InputManager.Current.PreProcessInput -= _handler;
           _timer.Stop();
       }

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Disk (auth.json) → in-memory master password verification | auth.json integrity depends on %AppData% ACL (per-user). Local attacker with AppData write can delete auth.json to reset (D-16 deliberate). |
| In-memory LockOverlayViewModel.Password → MasterPasswordService PBKDF2 | Plain string, scoped to unlock method; cleared immediately after SetMasterPassword / Verify (T-06-05 best-effort scrub). |
| SystemEvents thread → UI dispatcher | SessionSwitch fires on a dedicated thread; we marshal via Dispatcher.BeginInvoke (Pitfall 7). |
| WPF visual tree ↔ WindowsFormsHost HWND (airspace) | Lock overlay is WPF; RDP control is Win32 HWND. Pitfall 5 Option A: Visibility=Collapsed on every HostContainer child. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-03 | Spoofing / brute-force | MasterPasswordService.VerifyMasterPassword | mitigate | PBKDF2-HMAC-SHA256 @ 600,000 iterations (OWASP 2023 FIPS-path guidance); 32-byte random salt from RandomNumberGenerator.GetBytes; CryptographicOperations.FixedTimeEquals for compare (timing-attack mitigation). Iteration count enforced by Test 10 (Stopwatch ≥100ms floor). A1 documents the Argon2id tradeoff — accepted for v1 per CLAUDE.md constraint. |
| T-06-04 | Information Disclosure | Lock overlay bypass via airspace — RDP pixels visible through WPF overlay | mitigate | Pitfall 5 Option A: AppLockController.LockAsync snapshots every HostContainer child's Visibility and sets Collapsed; UnlockAsync restores per-child prior state. UAT file tests/uat/phase-06-security.md step 4 is the validation gate. If UAT reveals a visible leak, fall back to Option B (disconnect sessions on lock) — surfaces via UAT failure. |
| T-06-05 | Information Disclosure | Plaintext password in LockOverlayViewModel.Password while dialog open | accept | Project constraint (CLAUDE.md) bans SecureString (Microsoft deprecation DE0001). Best-effort mitigation: scope lifetime tightly (only during unlock method), clear immediately after VerifyMasterPassword / SetMasterPassword returns, never log, never serialise ([JsonIgnore] via the RedactSensitivePolicy denylist includes "Password" + "MasterPassword" — confirmed in Plan 06-01). |
| T-06-04b | Reliability | SystemEvents.SessionSwitch handler GC'd due to weak reference | mitigate | Pitfall 1 + Pattern 9: `_handler` stored in a strong-reference instance field on the DI-singleton SessionLockService; Dispose unsubscribes (verified by SessionLockServiceTests Test 4). |
| T-06-04c | Reliability | Idle-timer fires during RDP typing (would lock while user is working) | mitigate | Pitfall 6 filter: IdleLockService.HandleInput checks `FindAncestor<WindowsFormsHost>(Source)` and skips the reset when input source is inside an AxHost. D-14 intent preserved. |
| T-06-SEC-05 | Denial of Service | Missing unlock on Windows resume after suspend (Pitfall 4) | accept | DispatcherTimer does not "catch up" missed ticks. Acceptable because SessionSwitch on resume (SessionUnlock → our handler watches SessionLock/ConsoleDisconnect/RemoteDisconnect) typically lock the app on resume. Documented in tests/uat/phase-06-auto-lock.md. |
