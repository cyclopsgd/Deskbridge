# Phase 6 — UI Design Contract: Cross-Cutting Features

> Visual and interaction contract for the cross-cutting features phase (command palette, toast notifications, lock overlay, crash dialog, settings additions). Generated by gsd-ui-researcher, verified by gsd-ui-checker.
> This is a **WPF desktop application** using WPF-UI 4.2.0 (Fluent Dark). All values in device-independent pixels. All colours use `DynamicResource` tokens (or the project's existing `Deskbridge*Brush` resources) — never hardcoded `#RRGGBB` except in the three pre-existing custom brush definitions in `App.xaml`.
> Phase 6 **continues** the visual system established in Phases 2, 3, and 5. No new design tokens are introduced. The 4px spacing ramp, `12/14/16/20` type ramp with Regular/SemiBold weights, and 60/30/10 colour split all carry forward unchanged.
> Phase 6 adds four new modal/overlay surfaces (command palette, lock overlay, crash dialog) and one non-modal feedback surface (toast stack) — plus two fields inside the existing Settings panel shell for auto-lock configuration.

### Command Palette Internal Layout

```
ui:ContentDialog (DialogMaxWidth=480, DialogMaxHeight=480)
 Padding: 0 on dialog body (full-bleed; no outer title bar; no footer buttons)
 └ DockPanel (Width=480)
    ├ Top: SearchBox Border (height 48, padding 12)
    │      └ Grid
    │         ├ Col 0: SymbolIcon Search16 (16×16, muted)
    │         ├ Col 1: ui:TextBox (14px, no border, no icon, no clear button)
    │         └ Col 2: Shortcut hint "Esc" (12px, muted, right-padded 4)
    ├ Top: Thin divider (1px, ControlStrokeColorDefaultBrush)
    └ Fill: ListBox (Max Height 400 on content; scrolls on overflow)
           · Section header row — 28px, "RECENT CONNECTIONS" (11px uppercase muted)
           · Result rows — 36px each
           · Section header row — 28px, "COMMANDS"
           · Result rows — 36px each
```

**Positioning (D-01):** dialog is anchored near the top rather than perfectly centered. `ContentDialog` `VerticalAlignment="Top"` with `Margin="0,96,0,0"` on the dialog content element — 96px gap from window top feels "VS Code" / Spotlight. No horizontal offset — always centered horizontally.

**Backdrop:** WPF-UI default smoke overlay (dim). Clicking outside the palette closes it. Esc also closes.

### Command Palette Result Row Anatomy

Each row is 36px tall, built as a Grid with 4 named columns:

```
Row (36px, Padding=12,0)
 └ Grid
    ├ Col 0 (24px fixed):  Item icon (SymbolIcon 16×16)
    ├ Col 1 (*):           StackPanel
    │                        ├ TextBlock Title (14px Regular)
    │                        └ TextBlock Subtitle (12px TextFillColorSecondaryBrush, visible only if non-empty)
    ├ Col 2 (Auto):        Shortcut badge (shown only when shortcut exists)
    └ Col 3 (16px fixed):  right-edge breathing room
```

| Column | Value |
|--------|-------|
| Icon column width | 24px (16px icon + 8px right-padding) |
| Left row padding | 12px |
| Right row padding | 12px |
| Vertical text spacing | 0px (single-line) or 2px (title+subtitle) |
| Subtitle visibility | Only for **connection** rows where `Hostname != Name`; hidden for rows where hostname equals name (avoids redundant "srv01 / srv01"). Always hidden for command rows. |

### Section Separator (D-02)

Two visually distinct sections:

1. **RECENT CONNECTIONS** — top 5 connections by `LastUsedAt` descending (D-02). Always shown when input is empty, regardless of what's available.
2. **COMMANDS** — 4 items per D-04: New Connection, Settings, Disconnect All, Quick Connect.

Each section has a 28px header row:

```
Border (28px, Padding=12,8,12,4)
 └ TextBlock "RECENT CONNECTIONS" (11px, Regular, TextFillColorSecondaryBrush, all-caps)
```

When the user types a query, section headers are replaced by a single merged result list sorted by fuzzy score (no section grouping during search — D-03 dual-score + subsequence ranks commands and connections together).

### Lock Overlay Internal Layout

```
ContentDialog (fills entire client area — no DialogMaxWidth/Height; content fills)
 Background: ApplicationBackgroundBrush (opaque, D-15)
 └ Grid (centered both axes)
    └ Border (Width=360, Padding=32, CornerRadius=8)
       Background: ControlFillColorDefaultBrush
       BorderBrush: ControlStrokeColorDefaultBrush, BorderThickness=1
       └ StackPanel (vertical, spacing via per-element Margin)
          ├ TextBlock "Deskbridge" (20px SemiBold, centered, muted)
          ├ 8px gap
          ├ TextBlock body (14px, muted, centered — copy varies by mode)
          ├ 24px gap
          ├ ui:PasswordBox (40px tall, full width 296px, placeholder "Master password")
          ├ 8px gap (only when in first-run setup mode)
          ├ ui:PasswordBox #2 (only on first-run setup — "Confirm password")
          ├ 12px gap
          ├ TextBlock ErrorMessage (12px DeskbridgeErrorBrush, collapsed when empty)
          ├ 16px gap
          └ ui:Button Unlock (40px tall, full width, Appearance=Primary)
```

**Lock overlay is NOT a standard `ContentDialog` with Primary/Secondary footer buttons** — D-16 requires custom content with a single full-width Unlock button. Use a `ui:ContentDialog` subclass with `FooterVisibility="Collapsed"` (or equivalent — template override if needed per `WPF-UI-PITFALLS.md §1`), and place the Unlock button inside the dialog body StackPanel.

**Full-window opacity (D-15):** unlike command palette's dim backdrop, the lock overlay must fully hide any remote desktop pixels underneath. The ContentDialog's dim scrim is insufficient. Use an **opaque** background on the outer ContentDialog border (`ApplicationBackgroundBrush` at 100% opacity, not the semi-transparent scrim).

### Crash Dialog Internal Layout

```
ui:ContentDialog (DialogMaxWidth=480)
 Title: "Deskbridge encountered an unexpected error"
 Content: StackPanel
    ├ TextBlock body (14px, primary) — "An unexpected error occurred. A log file with details has been saved to your AppData folder. You can copy the error details to clipboard for support, or restart the application."
    └ (NO stack trace visible)
 PrimaryButton: "Copy Details"       (Appearance=Secondary)
 CloseButton:   "Restart"            (Appearance=Primary)
```

**Button role inversion (deliberate):** the Primary visual weight goes on **Restart** (the expected next action) rather than **Copy Details** (the assist action). WPF-UI's `ContentDialog` puts Primary on the left. `Restart` is mapped to the `CloseButton` slot with `CloseButtonAppearance="Primary"` if supported, or the buttons are placed manually in the dialog content if not.

### Toast Stack Layout

Already in place: `ui:SnackbarPresenter` at bottom-right (WPF-UI default anchor). Each `ui:Snackbar` is rendered by WPF-UI at 360px wide by default.

Per-toast anatomy (WPF-UI `ui:Snackbar` default template, unchanged):

```
ui:Snackbar (360px × auto)
 Background: ControlAppearance-dependent (Info / Caution / Danger / Success)
 Padding: 16 (WPF-UI default)
 ├ Icon column: 24×24 (SymbolIcon, appearance-tinted)
 ├ Content column:
 │  ├ Title (16px SemiBold)
 │  └ Message (12px Regular, muted)
 └ Close button: 16×16 Dismiss16 (top-right)
```

Max **3 visible simultaneously** (D-07). Newest on top. 4th push evicts oldest. Hover pauses auto-dismiss (WPF-UI default — no override needed).

## Typography

Inherited from Phases 2/3/5. **No new sizes, no new weights.**

| Role | Size | Weight | Line Height | Phase 6 Usage |
|------|------|--------|-------------|---------------|
| Caption | 12px | Regular (400) | 16px (1.33) | Toast body, palette subtitle, palette section header, settings helper text, lock error message, shortcut badge, shortcut hint in search row |
| Body | 14px | Regular (400) | 20px (1.43) | Palette search TextBox, palette result title, lock overlay body text, crash dialog body, settings field labels |
| Body Strong | 14px | SemiBold (600) | 20px (1.43) | **Command rows** in palette — palette commands render title in SemiBold to distinguish from connection rows (which use Regular). Also palette **selected row** regardless of type. |
| Subtitle | 16px | SemiBold (600) | 22px (1.375) | Toast title (WPF-UI default), crash dialog title (WPF-UI default) |
| Title | 20px | SemiBold (600) | 28px (1.4) | Lock overlay "Deskbridge" heading |

**Two weights only:** Regular (400) and SemiBold (600). No Bold, no Light, no italic.

**Selected-row typography** in palette: when a row is selected (keyboard navigation or hover+focus), the title text switches from Regular to SemiBold AND the foreground switches from `TextFillColorPrimaryBrush` to `TextOnAccentFillColorPrimaryBrush` (accent background). This dual change (weight + colour) is the single typography signal for "this is the selected row" — no font-size change, no underline.

**Monospace / code font:** not used in Phase 6. Hostnames in palette subtitle render in the same Segoe UI Variable as the rest of the UI.

### Toast Severity Visual Contract (D-06)

WPF-UI `ControlAppearance` maps to WPF-UI's built-in palette. Per-severity details:

| Severity | `ControlAppearance` | Icon (SymbolRegular) | Icon tint source | Auto-dismiss |
|----------|---------------------|---------------------|------------------|--------------|
| **Info** | `ControlAppearance.Info` | `Info24` | WPF-UI info palette (blue-tinted) | 2-3s per D-06 (connected=2s, disconnected=3s, updates-available=sticky) |
| **Warning** (reconnecting) | `ControlAppearance.Caution` | `ArrowClockwise24` | WPF-UI caution palette (amber — matches `DeskbridgeWarningBrush` #FFCC02) | Sticky until resolved |
| **Error** (failed) | `ControlAppearance.Danger` | `ErrorCircle24` | WPF-UI danger palette (red — matches `DeskbridgeErrorBrush` #F44747) | Sticky until dismissed |
| **Success** | *(not used in Phase 6 — see note above)* | — | — | — |

**Icon size:** 24px SymbolIcon (WPF-UI Snackbar default). Do not override.

**Icon name rationale:**
- `Info24` — standard info glyph, WPF-UI compatible (validated in Phase 3 editor dialog InfoBar).
- `ArrowClockwise24` — rotational arrow conveys "retrying". Chosen over `Warning24` (which reads as "attention needed" rather than "in progress"). Matches the reconnect overlay's semantic.
- `ErrorCircle24` — filled error circle, stronger visual weight than `Error24`. WPF-UI Danger default.

### Toast Copywriting (D-06)

All toast strings use the same format: `{Title}` + `{Message}`. The message body may interpolate `{Hostname}`, `{Attempt}`, `{Max}`, `{Version}`.

| Event | Severity | Title | Message | Auto-dismiss |
|-------|----------|-------|---------|--------------|
| `ConnectionEstablishedEvent` | Info | `Connected` | `Connected to {Hostname}.` | 2s |
| `ConnectionClosedEvent` (user-initiated) | *(no toast — intentional silence; closing is user-directed)* | — | — | — |
| `ConnectionClosedEvent` (remote disconnect / error) | Info | `Disconnected` | `Disconnected from {Hostname}.` | 3s |
| `ReconnectingEvent` | Warning | `Reconnecting` | `Connection to {Hostname} lost — attempt {Attempt}/20.` | Sticky until `ConnectionEstablishedEvent` or `ConnectionFailedEvent` fires, or user dismisses |
| `ConnectionFailedEvent` | Error | `Connection failed` | `Could not connect to {Hostname}. {Reason}.` | Sticky |
| `UpdateAvailableEvent` | Info | `Update available` | `Deskbridge {Version} is ready to install.` | Sticky |
| `ConnectionImportedEvent` | Info | `Import complete` | `Imported {Count} connection(s) from {Source}.` | 3s |

**Message punctuation rules:**
- All messages end with a period.
- Em-dash `—` (U+2014) for separators (e.g., "Connection to X lost — attempt 3/20").
- Middle-dot `·` not used in toast messages (reserved for status bar per Phase 5).
- `{Hostname}` renders unquoted (e.g., `Connected to srv01.example.com.` — not `"srv01.example.com"`).

**Deliberate silence:**
- User-initiated disconnect (tab close via X, Ctrl+W, etc.) does NOT emit a toast. User action doesn't need confirmation (D-08 "No modals for non-critical events" extends to "no toasts for user-directed actions").
- Successful auto-reconnect (after a Reconnecting warning) replaces the sticky Warning toast with an Info toast (`Reconnected to {Hostname}.` for 3s). The warning must be dismissed before the info appears.

| Event | Severity | Title | Message | Auto-dismiss |
|-------|----------|-------|---------|--------------|
| Reconnect success | Info | `Reconnected` | `Reconnection to {Hostname} succeeded.` | 3s |

## Interaction Contracts

### Command Palette (CMD-01..04)

**Open / close:**

| Gesture | Action |
|---------|--------|
| `Ctrl+Shift+P` | Open palette. If already open, no-op (idempotent). |
| `Esc` | Close palette. Restores focus to the previously active element. |
| Click outside palette (on dim backdrop) | Close palette. |
| Select a result (Enter, or click) | Execute the row's command/connection action, then close palette. |

**Keyboard navigation inside palette:**

| Gesture | Action |
|---------|--------|
| Down Arrow | Move selection down one row. Wraps from last → first. |
| Up Arrow | Move selection up one row. Wraps from first → last. |
| Page Down / Page Up | Move selection by 5 rows. |
| Home / End | Jump to first / last row. |
| Enter | Execute selected row. |
| Typing any character | Updates search query, replaces result list. Selection resets to top row. |

**Selection indicator (explicit):**

- Selected row title foreground: `TextOnAccentFillColorPrimaryBrush`.
- Selected row subtitle foreground: `TextOnAccentFillColorPrimaryBrush` at 0.8 opacity (subtitle is still visible, slightly softer than title).
- Selected row icon foreground: `TextOnAccentFillColorPrimaryBrush`.

**Result ranking (D-03):**
1. When `SearchText` is empty: show `RECENT CONNECTIONS` section (top 5 by `LastUsedAt` desc), then `COMMANDS` section.
2. When `SearchText` is non-empty: single merged list ranked by `IConnectionQuery.Search()` dual-score:
   - Connections score over `{Name, Hostname, Tags}` (existing Phase 3 surface).
   - Commands score over `{Title, Aliases, "command" pseudo-group}`.
   - Tied scores break by: commands first, then alphabetical title.

**Command registration (for research → planner):** commands are a static list held in a `CommandPaletteService`. The service provides `IEnumerable<ICommandEntry>` where `ICommandEntry` has `Title`, `Icon`, `Shortcut?`, `Subtitle?`, `Execute()`. ### Lock Overlay (SEC-01..05)

**Trigger mechanism:**

| Trigger | Action |
|---------|--------|
| App launch (always, after auth.json exists) | Show lock overlay immediately, before any other UI rendering. |
| App launch (first run, auth.json missing) | Show lock overlay in first-run setup mode. |
| `Ctrl+L` (SEC-04 / D-18) | Lock immediately. Idempotent — pressing while locked is no-op. |
| Auto-lock timer fires (SEC-03 / D-14, default 15 min) | Lock immediately. Timer resets on any Deskbridge-window mouse/keyboard input. |
| `SystemEvents.SessionSwitch` (SEC-04 / D-14) | Lock immediately. Covers Windows lock screen, user switch. |
| `Window.StateChanged` → `Minimized` (SEC-05 / D-19, default OFF) | Lock immediately if setting is ON. |

**Focus behaviour:**
- Lock overlay shows → password field gains focus automatically.
- Tab/Shift+Tab cycles: password field → (confirm field, if first-run) → Unlock button → password field (wraps).
- Enter in password field submits the unlock attempt (same as clicking Unlock).

**Unlock flow:**
1. User types password, presses Enter or clicks Unlock.
2. PBKDF2 hash computed, compared against stored hash.
3. Match: overlay dismisses, focus returns to the element that had focus before lock (or MainWindow default if lock came from an external trigger).
4. Mismatch: error message appears, password field is cleared and re-focused, button remains enabled for retry.

**Unlock animation:** none (instant snap, matching project pattern — no Phase 6 animations).

**First-run setup flow:**
1. User enters new password (≥8 chars).
2. User enters confirmation (must match exactly — byte-level comparison of the resulting strings).
3. User clicks `Set Password` or presses Enter in the confirm field.
4. Both fields validate: length ≥8, match. Errors render above the button.
5. On success: password is PBKDF2-hashed (parameters per research RESEARCH.md), stored to `%AppData%/Deskbridge/auth.json`, overlay dismisses, app proceeds normally.

### Toast Notifications (NOTF-01..03)

Already partially wired in Phase 5 (Snackbar for 15-session warning). Phase 6 **extends** the existing `ISnackbarService` usage to consume the event bus.

**Subscription surface:** a new `ToastSubscriptionService` (singleton, injected into composition root) subscribes to the 7 events listed in the Copywriting table. Each event maps 1:1 to an `ISnackbarService.Show()` call with the appropriate severity, icon, title, message, and duration.

**Stack eviction (D-07):**
- WPF-UI `SnackbarPresenter` handles stacking natively (max visible + FIFO eviction).
- 4th simultaneous toast evicts the oldest visible one.
- Sticky toasts (Warning/Error) count toward the max; a new toast can evict a sticky one if it's at the bottom of the stack.

**Hover pause:** default WPF-UI behaviour — hovering any toast pauses all auto-dismiss timers. Move pointer away → timers resume.

**Click-through behaviour:** toasts do not block interaction with underlying UI. Users can keep using the app while toasts are visible.

### Crash Dialog (D-11)

Triggered by global exception handlers:
- `AppDomain.CurrentDomain.UnhandledException`
- `Dispatcher.UnhandledException`
- `TaskScheduler.UnobservedTaskException`

**Dialog lifecycle:**
1. Handler catches exception, writes full trace to log file.
2. Posts to UI dispatcher to show `ui:ContentDialog` via `IContentDialogService.ShowAsync()`.
3. User clicks `Copy Details` → clipboard populated with structured dump. Dialog remains open.
4. User clicks `Restart` → `_host.StopAsync()`, then `Process.Start(Process.GetCurrentProcess().MainModule.FileName)`, then `Application.Current.Shutdown()`.
5. If user dismisses via X (title bar close): app stays running in degraded state. Surface second log line noting this.

## Non-Goals & Explicit Rejections

For the checker and auditor — these must NOT appear in Phase 6 implementation or plans:

6. **No new colour tokens, no new font sizes, no new weights, no new spacing values off the 4px ramp.**
7. **No animations** — palette shows/hides instantly, lock overlay shows/hides instantly, toasts use WPF-UI default fade (no override).
8. **No re-skinning of WPF-UI Snackbar** — use WPF-UI `ControlAppearance` enum as specified in D-06. Do not author custom Snackbar templates.
9. **No hardcoded hex literals** in new XAML. Three existing `App.xaml` brush definitions are the only permitted hex values.
10. **No Newtonsoft.Json** — `auth.json` and `settings.json` use `System.Text.Json` source generators (CLAUDE.md constraint).
11. **No plaintext password in XAML bindings or ViewModel properties exposed to data templates** (LOG-05 / D-12 — `Password` property on `LockOverlayViewModel` must be `[JsonIgnore]` and bound only to `ui:PasswordBox.Password` which is not a standard bindable property — use code-behind with a password accessor, NOT a public bindable string).
