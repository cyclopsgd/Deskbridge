Replace the existing tab DataTemplate in MainWindow.xaml (lines 249-335, inside ItemsControl.ItemTemplate) with a refactored version that implements all three requirements. The outer Border element must keep its existing attributes (Height="30", Cursor="Hand", ToolTip binding, ContextMenu, InputBindings) and add the following structural changes:

**TAB-01v2 -- Animated active tab distinction:**

1. Add `x:Name="TabBorder"` to the outer Border element. Replace the Border's inline `Background="Transparent"` with a named SolidColorBrush child element for Storyboard animation targeting (Phase 10 pattern):
```xml
<Border.Background>
    <SolidColorBrush x:Name="TabBrush" Color="Transparent" />
</Border.Background>
```

2. Move the Border's inline Style triggers from `Style.Triggers` to `DataTemplate.Triggers` (required because DataTemplate.Triggers can target named child elements across the template via Storyboard.TargetName, while Style.Triggers cannot). The Border still needs a Style but only for the default BorderBrush Setter:
```xml
<Border.Style>
    <Style TargetType="Border">
        <Setter Property="BorderBrush" Value="Transparent" />
    </Style>
</Border.Style>
```

3. Add hover animation as a DataTemplate-level Trigger (SourceName="TabBorder" targeting "TabBrush"):
```xml
<Trigger SourceName="TabBorder" Property="IsMouseOver" Value="True">
    <Trigger.EnterActions>
        <BeginStoryboard x:Name="HoverEnter">
            <Storyboard>
                <ColorAnimation Storyboard.TargetName="TabBrush"
                    Storyboard.TargetProperty="Color"
                    To="{StaticResource SubtleFillColorSecondary}"
                    Duration="0:0:0.15" />
            </Storyboard>
        </BeginStoryboard>
    </Trigger.EnterActions>
    <Trigger.ExitActions>
        <BeginStoryboard x:Name="HoverExit">
            <Storyboard>
                <ColorAnimation Storyboard.TargetName="TabBrush"
                    Storyboard.TargetProperty="Color"
                    To="Transparent"
                    Duration="0:0:0.15" />
            </Storyboard>
        </BeginStoryboard>
    </Trigger.ExitActions>
</Trigger>
```

4. Add active tab animation as a DataTemplate-level DataTrigger (AFTER the hover Trigger so it takes precedence per WPF trigger ordering):
```xml
<DataTrigger Binding="{Binding IsActive}" Value="True">
    <DataTrigger.EnterActions>
        <BeginStoryboard x:Name="ActiveEnter">
            <Storyboard>
                <ColorAnimation Storyboard.TargetName="TabBrush"
                    Storyboard.TargetProperty="Color"
                    To="{StaticResource SubtleFillColorSecondary}"
                    Duration="0:0:0.12" />
            </Storyboard>
        </BeginStoryboard>
    </DataTrigger.EnterActions>
    <DataTrigger.ExitActions>
        <BeginStoryboard x:Name="ActiveExit">
            <Storyboard>
                <ColorAnimation Storyboard.TargetName="TabBrush"
                    Storyboard.TargetProperty="Color"
                    To="Transparent"
                    Duration="0:0:0.15" />
            </Storyboard>
        </BeginStoryboard>
    </DataTrigger.ExitActions>
    <Setter TargetName="TabBorder" Property="BorderBrush" Value="{DynamicResource SystemAccentColorPrimaryBrush}" />
</DataTrigger>
```

5. The TextBlock style triggers for active tab (Foreground to TextFillColorPrimaryBrush, FontWeight to SemiBold) remain as-is inside the TextBlock's inline Style -- they do not need animation.

**TAB-02v2 -- Hover-reveal close button:**

1. Give the existing close `ui:Button` the name `x:Name="CloseBtn"` and set its default `Opacity="0"`.

2. Add a DoubleAnimation to the hover Trigger's EnterActions Storyboard (alongside the TabBrush ColorAnimation) to fade the close button in:
```xml
<DoubleAnimation Storyboard.TargetName="CloseBtn"
    Storyboard.TargetProperty="Opacity"
    To="1" Duration="0:0:0.15" />
```

3. Add a DoubleAnimation to the hover Trigger's ExitActions Storyboard to fade the close button out:
```xml
<DoubleAnimation Storyboard.TargetName="CloseBtn"
    Storyboard.TargetProperty="Opacity"
    To="0" Duration="0:0:0.15" />
```

4. In the active DataTrigger, add a Setter to keep the close button always visible on the active tab:
```xml
<Setter TargetName="CloseBtn" Property="Opacity" Value="1" />
```

5. The close button keeps its existing Width="16", Height="16", Margin="4,0,0,0", Padding="0", Appearance="Transparent", Command binding, and CommandParameter binding exactly as they are. No layout shift occurs because the button stays in the layout flow (Opacity change does not affect layout).

**TAB-03v2 -- Connection state tint overlay:**

1. Replace the Border's content from a direct StackPanel child to a Grid wrapper containing two layers:
```xml
<Grid>
    <!-- State tint overlay (behind content, not competing with hover animation) -->
    <Rectangle x:Name="StateTint" Opacity="0.05" IsHitTestVisible="False">
        <Rectangle.Style>
            <Style TargetType="Rectangle">
                <Setter Property="Fill" Value="Transparent" />
                <Style.Triggers>
                    <DataTrigger Binding="{Binding State}" Value="Connected">
                        <Setter Property="Fill" Value="{DynamicResource SystemAccentColorPrimaryBrush}" />
                    </DataTrigger>
                    <DataTrigger Binding="{Binding State}" Value="Reconnecting">
                        <Setter Property="Fill" Value="{DynamicResource DeskbridgeWarningBrush}" />
                    </DataTrigger>
                    <DataTrigger Binding="{Binding State}" Value="Error">
                        <Setter Property="Fill" Value="{DynamicResource DeskbridgeErrorBrush}" />
                    </DataTrigger>
                </Style.Triggers>
            </Style>
        </Rectangle.Style>
    </Rectangle>
    <!-- Existing content (StackPanel with indicators + title + close button) -->
    <StackPanel Orientation="Horizontal" VerticalAlignment="Center">
        ... (existing content unchanged)
    </StackPanel>
</Grid>
```

2. The Rectangle sits at Grid position (0,0) behind the StackPanel (WPF renders in document order; the StackPanel on top receives all hit events). Opacity="0.05" gives a barely-perceptible tint. IsHitTestVisible="False" ensures the overlay does not interfere with clicks.

3. The Connecting state gets no tint (the ProgressRing spinner is sufficient visual feedback). Disconnected also has no special tint (default Transparent fill).
