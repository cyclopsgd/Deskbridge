## Component Inventory

All components listed below exist in WPF-UI or in the project's shared resource dictionaries. No new custom controls are created.

### New Settings Sections (2)

**1. BULK OPERATIONS card**

```
[SectionLabelStyle]  BULK OPERATIONS
[CardContainerStyle border]
  [BodyStyle]        Confirm before bulk operations
  [ToggleSwitch]     IsChecked=true (default: on)
  [SeparatorStyle]   ────────────────────────────────
  [FieldLabelStyle]  GDI handle warning threshold
  [NumberBox]        Value=15, Min=5, Max=30
[/border]
```

- Uses `SeparatorStyle` border between the two controls (per Claude's Discretion area in D-03 of CONTEXT.md)
- NumberBox matches the existing auto-lock timeout NumberBox pattern in the Security section
- NumberBox clamps out-of-range values on LostFocus (WPF-UI default behavior -- no custom validation needed)

**2. UNINSTALL card**

```
[SectionLabelStyle]  UNINSTALL
[CardContainerStyle border]
  [BodyStyle]        Clean up application data on uninstall
  [ToggleSwitch]     IsChecked=false (default: off / preserve data)
  [HintStyle]        Removes connections, credentials, logs, and
                     settings from %AppData% during uninstall.
[/border]
```

- Description text uses `HintStyle` (12px, `TextFillColorTertiaryBrush`) for muted inline explanation
- TextWrapping="Wrap" on the description TextBlock to handle the 240px panel width

### Section Order (top to bottom, per D-02)

1. APPEARANCE (existing)
2. SECURITY (existing)
3. BULK OPERATIONS (new)
4. DATA (existing -- moves from position 3 to position 4)
5. UNINSTALL (new)

### Insertion Point

New sections are added in `MainWindow.xaml` within the settings `StackPanel` (currently lines 217-300). BULK OPERATIONS inserts between SECURITY and DATA. UNINSTALL appends after DATA.

## Interaction States

### ToggleSwitch (both "Confirm before bulk operations" and "Clean up application data on uninstall")

| State | Behavior |
|-------|----------|
| Default | Off-track/on-track per initial value. WPF-UI Fluent styling. |
| Hover | WPF-UI built-in subtle fill on the switch track |
| Focus | WPF-UI built-in focus rectangle (keyboard navigation) |
| Disabled | Not applicable -- both toggles are always enabled |
| Changed | Triggers `SaveSettingsAsync()` immediately (D-11 auto-save pattern) |

### NumberBox (GDI handle warning threshold)

| State | Behavior |
|-------|----------|
| Default | Displays "15" |
| Focus | WPF-UI input active styling (accent bottom border) |
| Out-of-range input | WPF-UI NumberBox clamps to Minimum (5) or Maximum (30) on LostFocus |
| Empty input | WPF-UI NumberBox reverts to previous valid value on LostFocus |
| Changed | Triggers `SaveSettingsAsync()` on LostFocus (matches auto-lock timeout pattern) |
| Spinner buttons | WPF-UI NumberBox includes built-in increment/decrement buttons |

## Copywriting Contract

| Element | Copy |
|---------|------|
| Section header 1 (new) | `BULK OPERATIONS` |
| Section header 2 (new) | `UNINSTALL` |
| Bulk confirm label | `Confirm before bulk operations` |
| GDI threshold label | `GDI handle warning threshold` |
| Uninstall toggle label | `Clean up application data on uninstall` |
| Uninstall description | `Removes connections, credentials, logs, and settings from %AppData% during uninstall.` |
| Primary CTA | Not applicable -- no action buttons in this phase (auto-save on change) |
| Empty state | Not applicable -- settings panel always renders its full content |
| Error state | Not applicable -- no validation errors possible (NumberBox clamps, toggles are binary) |
| Destructive confirmation | Not applicable -- this phase persists a preference only; Phase 24 implements the destructive uninstall action |

### Copywriting Notes

1. **Section headers** follow the existing UPPERCASE convention: `APPEARANCE`, `SECURITY`, `BULK OPERATIONS`, `DATA`, `UNINSTALL`.
2. **Toggle labels** use sentence case matching the existing `Require password/PIN` and `Lock on minimise` patterns.
3. **Uninstall description** omits the "Default: data is preserved" sentence from D-07. The default off-state makes preservation implicit; including "Default:" text creates a contradiction when the user enables the toggle. The description should state what the feature does, not its initial value. If the user prefers the original wording, it can be restored.
4. **No ellipsis** on any label -- none of these controls open a follow-up dialog.

## Persistence Contract

| Setting | AppSettings Path | Type | Default | Auto-Save Trigger |
|---------|-----------------|------|---------|-------------------|
| Confirm before bulk operations | `BulkOperations.ConfirmBeforeBulkOperations` | `bool` | `true` | ToggleSwitch.IsChecked changed |
| GDI warning threshold | `BulkOperations.GdiWarningThreshold` | `int` | `15` | NumberBox.Value changed (LostFocus) |
| Clean up on uninstall | `Uninstall.CleanUpOnUninstall` | `bool` | `false` | ToggleSwitch.IsChecked changed |

New records follow the existing nullable sub-record pattern (D-10):
- `BulkOperationsRecord?` on `AppSettings`, null-coalesced to `BulkOperationsRecord.Default` on load
- `UninstallRecord?` on `AppSettings`, null-coalesced to `UninstallRecord.Default` on load
- `AppSettingsContext` must add `BulkOperationsRecord` and `UninstallRecord` to its JSON source generation
